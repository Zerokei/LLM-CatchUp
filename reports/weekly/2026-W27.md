## 分类概览

### 模型发布

本周模型发布由 Anthropic 主导。Claude Sonnet 5 强化中高复杂度代理任务，Fable 5 则在重新上线后围绕安全分类器、访问规则和用量限制重新定义可用性。Berkeley RDI 对 GPT-5.5-Cyber 的关注说明网络安全模型仍是前沿能力竞争的重要方向。整体看，模型发布已经明显和安全、容量、产品入口以及政府协作交织在一起。

### 研究

研究线索一边进入生命科学，一边进入神经接口。OpenAI 的 GeneBench-Pro 把评测重点放在科研判断、证据边界和分析路径选择上，而不是简单答案。Meta Brain2Qwerty v2 则把非侵入式脑信号实时解码推进到词语和语义级句子。两者共同说明，AI 研究正在转向更专业、更难验证、也更需要开放数据和可审计流程的任务。

### 产品与功能

产品更新最集中在科研工作台、企业云入口、语音代理和开发工具。Claude Science 把科研代理做成可连接 HPC 与远程环境的工作台，Claude in Microsoft Foundry 则把 Anthropic 模型带进 Azure 治理边界。xAI Voice Agent Builder、Grok 语音 API、Claude Code Artifacts、ZCode、Notion HTML 块和 Google 6 月更新共同显示：产品层正在把语音、交互页面、代码代理和文档原型快速商品化。

### 商业动态

商业动态数量不高，但都围绕分发和生态入口。Anthropic 借 Microsoft Foundry 扩展企业采用路径，xAI 借 Vercel、Railway 等开发者平台扩大 Grok 的部署场景，Z.ai 则用 ZCode 为 GLM-5.2 建官方开发环境。模型公司的商业竞争正在从“谁的模型更强”延伸到“谁能嵌入企业采购、开发者工具和云沙箱”。

### 政策与安全

政策与安全的核心是 Anthropic 的 Fable 5 重新上线。出口管制解除、安全分类器修复、越狱严重性框架和政府预发布测试合作组成了一套更正式的发布治理流程。对开发者而言，安全治理不再只是外部政策新闻，而会影响模型是否可用、请求是否回退、额度是否重置以及误拦截如何反馈。

### 教程与观点

教程与观点显著增加，说明行业开始讨论如何组织 AI 辅助开发和全栈 AI 平台，而不只是追逐单个发布。Google 的 full-stack AI 解释、Spotify 的 73% PR AI 辅助案例、OpenAI 的核心转储排障复盘，都把注意力拉回工程流程、可靠性和平台结构。它们提醒团队：AI 工具能提高产出，但真正的瓶颈常常在测试、调试、治理和组织反馈回路。

## 本周值得上手试试

1. **Claude Sonnet 5** — 在现有 Claude Code 工作流中复测多步骤修复、重构和浏览器工具任务，并比较 Sonnet 5 与 Opus 4.8 的成本成功率。
2. **Fable 5** — 重新跑核心编码、长文档和安全边界测试，记录新分类器下的误拦截和回退情况。
3. **Claude Science** — 选择一个已有科研分析项目，在 Claude Science 中复现实验流程并检查审计历史是否完整。
4. **Claude in Microsoft Foundry** — 在 Azure 中测试 Claude Messages API 的认证、计费、网络策略、prompt caching 和 extended thinking。
5. **xAI Voice Agent Builder** — 用同一客服脚本对比现有 STT-LLM-TTS 组合与 Grok Voice 的延迟、成本和失败率。
6. **Google 6月更新** — 挑选 Gemma 4 12B、Gemini Live Translate 或 NotebookLM 升级中最贴近产品的能力做小规模验证。
7. **Claude Code Artifacts** — 用 Artifacts 做一个 PR walkthrough、项目仪表盘或临时交互原型，并评估私有链接的协作边界。
8. **ZCode** — 在小型代码库中试用 ZCode，验证 GLM-5.2 的补全、修改、长上下文和 BYOK 配额体验。
9. **Notion HTML 块** — 把一份静态规格文档改造成可点击原型或交互式流程图，测试团队共享后的权限边界。
