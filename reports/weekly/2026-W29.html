分类概览

模型发布

本周模型发布共有 5 篇，竞争焦点从单一参数规模转向代码、前端、长上下文和真实任务表现。Kimi K3 以开放模型、百万上下文和视觉闭环编程形成高辨识度发布叙事，Qwen3.8 则提前释放 Max Preview 并预告开放权重。Grok 4.5 的 FrontierSWE 表现和 GPT-5.6 Sol 的安全能力展示，进一步推动模型比较进入工程任务和专业领域。

研究

研究类共有 8 篇，安全与科学应用占据主要位置。GPT-Red 用自动红队生成提示注入攻击，Anthropic 的代理错位模拟则把自主系统风险放进可复核的场景转录。与此同时，医疗、物理竞赛和加拿大区域采用研究显示，模型评估正逐步覆盖专业工作与社会影响，而不只依赖通用基准。

产品与功能

产品与功能共有 28 篇，是本周最活跃的类别。Codex 继续扩展到并行目标、Sites、Inline edits、移动端和 SSH；Claude Code 则围绕代码审查、Managed Agents、无障碍和套餐能力补齐工作流。Google Search AI Mode、Claude Artifacts MCP 连接器和 Claude for Teachers 共同表明，产品竞争正在从聊天界面转向权限、数据和真实协作流程。

商业动态

商业动态共有 3 篇，整体以组织扩张、合作和市场信号为主，结构性交易较少。OpenAI 的企业采用与 Codex 活跃度、Anthropic 的研究投入以及 Kimi 的订阅容量变化，分别体现需求增长、生态建设和算力约束。对使用者而言，服务稳定性和单位任务成本比单纯的用户规模更值得持续跟踪。

政策与安全

政策与安全共有 4 篇，重点从抽象原则转向部署中的可观测风险。GPT-Red、代理错位模拟和 Codex Security 都强调在真实工具链、网页、邮件和代码环境中验证攻击面。教育产品的数据隐私、OpenAI 的青少年安全路线和生物韧性讨论，则把治理要求延伸到具体产品和社会基础设施。

教程与观点

教程与观点共有 17 篇，主要围绕如何把代理能力转化为可控的工作流。Thariq 关于精简系统提示词、Codex Build Week 和社区项目展示了从提示、原型到交付的实践路径。与此同时，关于模型就业影响、Token 成本和研发流程的讨论提醒团队：评估 AI 应优先看任务完成成本、返工率和长期协作质量。
