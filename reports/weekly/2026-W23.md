# CatchUp 周报 — 2026-W23 (06/01 - 06/07)

## 本周趋势

- **代理模型竞争**：Qwen3.7-Plus 与 Grok Build Composer 2.5 同周更新，模型发布开始直接围绕浏览器操作和构建工作流展开。

## 分类概览

### 模型发布

本周模型发布数量不多，但主题非常集中：模型不再只强调聊天或基准，而是直接面向代理执行。Qwen3.7-Plus 主打多模态代理和浏览器操作能力，把视觉理解、网页状态和工具调用放到同一套流程中检验。xAI 的 Grok Build Composer 2.5 则嵌在构建工具内发布，强调开发者可以直接在 CLI 和模型菜单中试用。两条新闻共同说明，模型发布正在更紧地绑定具体工作台和操作环境。

### 研究

研究板块的核心是 Google DeepMind 的 Gemini for Science。它把 Gemini 包装成多智能体科研助手，覆盖假设生成、文献推理和实验设计，而不是停留在通用问答增强。Co-Scientist 的演示显示，AI 科研工具正在从“帮我解释论文”走向“帮我组织研究路径”。本周历史窗口里研究文章较少，因此这一条几乎独占了该分类的主线。

### 产品与功能

产品与功能继续由编码和工作流平台主导。OpenAI 把 Codex 扩展到角色插件、Codex Sites、团队批注和 AWS 分发，试图让非工程岗位也能使用 Codex 生成可审阅产物。Claude Code 则通过后台代理、CLI 和托管会话连接，把本地开发任务拆成更可并行的子流程。OpenAI Devs 的 Voice Hack Night 原型补上了语音 Agent 方向，说明移动端和语音壳层仍在寻找新的交互入口。

### 商业动态

商业动态本周主要体现为客户案例和行业落地。Travelers 使用 OpenAI Realtime API 和前沿模型构建车险理赔语音助手，两个月内从 8 个州扩展到全美，且使用该助手的客户中有 85% 到 90% 完成理赔提交。Claude 展示 Legora 法律工作流案例，强调专业服务中的文档、检索和协作流程。两者都不是平台级发布，但说明 AI 落地正在进入保险和法律这类流程重、合规重的行业。

### 政策与安全

政策与安全是本周最密集的分类之一。Anthropic 扩大 Project Glasswing 至 150 个组织，并公开支持美国 AI 行政令，显示其政策参与方式从表态走向合作网络建设。OpenAI 则同时推动国际青少年 AI 安全机制和 Foundation 韧性资助，覆盖年龄识别、家长控制、独立审计、生物韧性、网络韧性和模型安全。整体看，头部公司正在把安全治理拆成可资助、可审计、可外交协调的制度化项目。

### 教程与观点

教程与观点较轻量，主要是提示词和使用观察。Thariq 转述 Anthropic 员工关于 Claude 跟进提示的方法，强调一次回答之后继续追问约束、改写和验证，适合沉淀为团队提示词手册。另有语音模式用于更自然回复的观察，以及 Sam Altman 分享个人座右铭。该分类本周缺少高影响长文，但补充了日常使用层面的细碎经验。

## 本周值得上手试试

1. **Qwen3.7-Plus 浏览器代理** — 用表单填写、网页检索和多步骤后台操作做一组同题测试，重点记录视觉理解、工具调用失败和页面状态恢复能力。
2. **Claude Code /fork 后台代理** — 在当前开发上下文中用 `/fork` 拆出一个独立子任务，观察后台代理是否能减少等待和上下文切换。
3. **Codex 角色插件与 Sites** — 为销售、研究或产品运营各设计一个小型 Codex 插件原型，再用 Codex Sites 生成可批注的共享页面。
4. **Codex on AWS / Bedrock** — 检查现有 AWS 账户能否访问 OpenAI 模型和 Codex，用一个小型集成验证权限、审计、账单和延迟差异。
5. **Grok Build Composer 2.5** — 从 `x.ai/cli` 进入 Grok Build，在同一个原型任务上对比 Composer 2.5 与现有代码/界面生成工具的多轮修改质量。
6. **语音 Agent 原型** — 选一个手机端高频任务，把点击流程改写成语音优先流程，参考 Voice Hack Night 项目验证指令理解和动作确认体验。
