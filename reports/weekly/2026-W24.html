本周趋势

  • 前沿模型放量：Anthropic以Claude Fable 5开放Mythos级能力，Google则用DiffusionGemma探索文本扩散生成。
  • 安全回退显性化：Claude把高风险请求回退到Opus 4.8并公开拒答原因，随后又遇到美国出口管制访问暂停。

  • 企业采用深化：BBVA、LSEG、DXC和Nextdoor案例显示，AI部署正从个人提效进入组织流程与核心系统。

分类概览

模型发布

本周模型发布由Claude Fable 5主导，Anthropic把Mythos级能力推向更广用户，同时用可信访问和回退机制给高风险领域设边界。Google DeepMind则在两个方向补充：Gemini 3.5 Live Translate把多语实时音频翻译推向应用，DiffusionGemma用文本扩散尝试更快、更可修正的开放生成模式。整体看，模型能力提升正和产品入口、安全分类、成本路由一起发布。模型厂商不再只讲“更强”，而是讲“谁能用、何时回退、如何接入”。

研究

研究线索集中在AI系统进入真实复杂环境后的基础设施问题。Anthropic关于生物智能体的研究指出，科学代理要可靠工作，需要确定性工具和可机器执行的数据层，而不是只依赖更强推理。Google DeepMind在教育、足球战术和多智能体基金上的动作，则展示AI研究从单模型能力转向学习行为、空间决策和群体互动。值得注意的是，多智能体生态的风险研究已开始被提前资助，说明“许多代理同时运行”正在成为新研究对象。

产品与功能

产品更新几乎都围绕代理工程化和开发者平台展开。Claude连接器仪表盘、提交通道、Managed Agents定时部署、vault环境变量、dynamic workflows和Apple Foundation Models支持，构成一套从接入、观测到自动运行的工程闭环。OpenAI Codex则补上速率重置、浏览器调试和文档代理等开发体验细节，xAI的Grok Build插件市场也把数据库、部署和监控工具拉入代理环境。本周的共同方向是：代理要真正进入生产，就必须拥有凭证、沙箱、监控、调度和外部工具链。

商业动态

商业动态显示企业AI正在从试点扩展到组织级部署。OpenAI计划收购Ona，是为了强化Codex在客户控制基础设施中的持久云执行能力；BBVA、LSEG和Nextdoor案例则分别展示银行、金融数据和社区产品工程中的规模化使用。Anthropic与DXC的联盟把Claude带入银行、航空、保险、制造和政府等受监管行业，也说明企业服务商正在把前沿模型封装进托管服务与系统现代化项目。商业竞争焦点因此从模型API转向安全执行环境、采购路径、行业交付和认证人才。

政策与安全

政策与安全是本周最剧烈的副线。OpenAI披露疑似涉华影响力行动，显示生成式AI正在被用于围绕数据中心、电价、关税和科技竞争制造叙事攻击。Anthropic方面，Dario Amodei提出前沿AI强制第三方测试、灾难性风险模型部署阻止权和就业冲击政策框架；而美国随后要求暂停外国国民访问Fable与Mythos，更让前沿模型访问控制变成现实问题。透明度、测试、出口管制和模型回退正在合并成同一个治理议题。

教程与观点

教程与观点类文章主要回答“更强模型如何改变工作流”。OpenAI的Nextdoor与Notion案例强调，团队应从反复提示代理转向先定义目标、设计工具描述、评估真实任务，再让中心推理模型协调子代理。Karpathy和Thariq围绕Fable 5的观察，则把这种变化说得更直接：开发者的工作重心从检查模型有没有做完，转向判断它是否在做正确的事。OpenAI Academy的新课程和Codex科学计算案例也说明，AI素养、可验证研究和代理工作流正在成为组织培训内容。

本周值得上手试试

  1. Claude Fable 5 / Claude Code — 升级Claude Code CLI到最新版本，在一个非生产仓库运行/model claude-fable-5，用它完成一次多文件重构并重点检查任务分解是否正确。
  2. Claude拒答回退机制 — 阅读拒答处理文档，在测试项目中接入SDK中间件，模拟高风险或误报请求，确认应用能优雅回退到Opus 4.8而不是中断流程。
  3. Claude连接器观测仪表盘 — 如果已有MCP连接器，查看活跃用户、工具调用、错误率和延迟；如果没有，先选一个内部数据源做最小连接器并通过新提交通道验证流程。
  4. Claude Managed Agents — 用定时部署创建一个每周运行的小代理，例如自动检查依赖更新或生成项目状态摘要，并用环境变量凭证连接外部CLI或API。
  5. Gemini 3.5 Live Translate — 在Google Translate或AI Studio API预览中测试一段跨语言语音流，重点观察语气、节奏和专有名词保留效果。
  6. Grok Build插件市场 — 选择MongoDB、Vercel或Sentry插件做一次端到端实验：从数据探索、应用生成到部署或错误定位，评估插件链路是否减少上下文切换。
  7. OpenAI Codex浏览器调试 — 在一个前端Bug上尝试Codex的浏览器调试/CDP工作流，让代理复现页面问题并提交修复，再人工审查补丁。
  8. OpenAI文档代理 — 用开发目标生成一份定制指南，复制为Markdown给团队评审，检查它是否能替代手写接入说明的第一版。

值得深读
