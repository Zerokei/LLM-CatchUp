## 本周趋势

- **科研评测变真实**：OpenAI 的部署仿真、LifeSciBench 与 AI 化学家案例都把评测推进到生产请求和科研工作流。

## 分类概览

### 模型发布

本周模型发布数量不多，但方向很集中：开放权重、代码加速和具身世界建模。GLM-5.2 开源权重对中文、本地部署和开发者生态很重要，Kimi K2.7 Code HighSpeed 则用吞吐作为代码模型卖点。Qwen-Robot Suite 与 RobotWorld 更值得长期关注，因为它们把语言、视频世界模型和机器人动作接口放到同一条线上。模型发布正在从聊天能力转向可执行任务、物理控制和本地可部署性。

### 研究

研究是本周主轴，覆盖模型安全、经济影响、生命科学、医疗和机器人。OpenAI 的部署仿真把真实请求引入发布前风险预测，LifeSciBench 和 AI 化学家案例把科研评测推向完整工作流。Anthropic 的 Claude Code 经济研究说明专业知识仍决定代理使用成效，Project Fetch 则探索通用模型能否帮助非专家做机器人任务。整体看，行业开始用更贴近真实世界的任务来衡量模型，而不是只依赖静态基准。

### 产品与功能

产品更新主要围绕企业权限、设计协作、健康问答和开发者入口。Claude MCP 企业托管认证把连接器权限下沉到协议层，Artifacts 与 Claude Design 双向同步让 Claude Code 更适合团队交付。OpenAI 的 ChatGPT 健康智能、企业用量控制和 Codex 插件分别补齐安全问答、成本治理和开发者配置。xAI Grok on Bedrock、Warp、DigitalOcean Marketplace 也显示模型能力正在快速嵌入现有云和终端环境。

### 商业动态

商业动态较少，但三个信号都与基础设施和人才有关。Google 加码阿拉巴马数据中心，说明 AI 基础设施扩张仍在继续向能源、社区和长期资本支出延伸。OpenAI 支持 Rust Foundation 和维护者生态，是其开源供应链策略的一部分。Noam Shazeer 加入 OpenAI 则是重要人才流动信号，可能加强 OpenAI 在模型架构和产品化研究上的经验储备。

### 政策与安全

政策与安全的核心来自 Google DeepMind 的 AI Control Roadmap。它不假设高级 AI 总会按人类意图行动，而是提前规划误解、过度执行和多智能体失控情形。结合 OpenAI 的部署仿真，本周的安全主题很明确：在代理系统规模化前，实验室需要把真实流量、工具状态和多层控制协议纳入发布前评估。

### 教程与观点

本周没有明显独立的教程与观点类文章，但许多研究和产品更新本身已经给出实践路线。Claude Code 经济研究告诉团队要沉淀领域词表和验收证据，部署仿真提示模型发布前要使用代表性流量，Claude MCP 企业认证则提示权限治理应前移到协议层。对实践者来说，本周更像一组“如何评测、治理和组织代理工作”的方法论更新。

## 本周值得上手试试

1. **GLM-5.2** — 下载权重并用内部中文任务集、工具调用和代码任务做本地基线评测。
2. **Kimi K2.7 Code HighSpeed** — 用团队真实代码补全、重构和 CI 辅助审查任务测试吞吐与质量。
3. **Deployment Simulation** — 设计一个去标识化生产请求回放流程，用于候选模型上线前风险评估。
4. **Claude Code 经济研究方法** — 按岗位、任务类型、领域词汇和成功条件统计内部 Claude Code 使用效果。
5. **Qwen-Robot Suite** — 查看官方 demo，挑选导航或操作任务在仿真环境做小规模复现。
6. **AI 化学家工作流** — 把文献综述、假设生成、实验设计和验证拆成可审计步骤，评估模型在每步的作用。
7. **Grok 4.3 on Bedrock** — 在 Bedrock 中比较 Grok 与现有模型的工具调用可靠性、权限隔离、延迟和成本。
8. **Claude Design / Code 同步** — 用 `/design-sync` 在组件库仓库试跑设计系统同步，检查 token、组件属性和实现是否能稳定往返。
9. **Claude MCP 企业托管认证** — 评估现有 MCP 连接器是否适合集中授权，并检查撤权、审计和团队分层权限。
