分类概览

模型发布

本周模型发布的主线是代理化和分层化。OpenAI 一方面推出安全导向的 Daybreak 与 GPT-5.5-Cyber，另一方面预览 GPT-5.6 Sol/Terra/Luna 的能力与成本分层。字节 Seed2.1、Qwen-AgentWorld 和 GPT-5.5 Instant 则分别从生产力、多环境智能体和日常对话入口强化模型。整体看，模型发布越来越少是单一权重更新，更多是模型、评测、工具、安全和产品入口的组合包。

研究

研究类内容集中在模型进入真实复杂任务后的验证问题。GPT-5 Pro 的免疫学案例展示模型能帮助科学家重构实验假设，但也让生物安全与实验复核更重要。OpenAI 关于 Codex 与知识工作的研究显示，代理式 AI 正把工作单位从一次性对话变成可委派任务。Anthropic 经济影响研究则开始量化 AI 使用节奏、岗位预期和自动化程度之间的关系。

产品与功能

产品更新围绕团队代理、文档处理和开发环境展开。Claude Tag 是最重要的产品信号，它把 Slack、长期记忆、权限和异步执行放进同一个团队工作流。Mistral OCR 4、Gemini computer use、Codex 移动端和 DigitalOcean 插件分别补齐文档结构化、界面操作、移动跟进和云端持久开发环境。共同方向是让代理更接近真实组织流程，而不是停留在聊天框。

商业动态

商业动态的焦点是基础设施和平台控制权。OpenAI 与 Broadcom 的 Jalapeño 推理芯片显示模型公司希望掌控成本、延迟和容量曲线。Sam Altman 关于 token 供给的表态也强化了这一点：当用户和代理吞吐继续上升，推理基础设施会直接影响产品形态和价格策略。企业采用方面，Claude Tag 和 Codex 扩展都在把 AI 从个人工具推向团队级采购与运营。

政策与安全

政策与安全围绕开源安全、可信访问和防御能力展开。OpenAI 的 Patch the Planet 与 Codex for Open Source 说明模型公司正在把安全投资投向关键依赖维护者。Daybreak Cyber Partner Program 又把 GPT-5.5-Cyber 放进受控访问和行业伙伴生态。相比单纯发布安全模型，本周更重要的是修复闭环、权限边界和人工审查流程开始被产品化。

教程与观点

教程与观点数量不多，但方向很明确：如何把代理工作流接入现有团队。Claude Tag 的内部实践、Codex 移动端和云环境案例都提示团队需要重新设计任务分派、审阅、通知和权限边界。对开发者来说，本周值得学习的不是某个提示词，而是如何让代理在 Slack、移动端、云 IDE 和代码审查之间持续工作。

本周值得上手试试

  1. OpenAI Daybreak / Codex Security — 选一个授权代码库做安全扫描试点，要求输出漏洞证据、严重性、补丁建议和测试结果。
  2. Seed2.1 — 用多步骤办公、GUI 操作或端到端代码交付任务复测 Seed2.1，重点记录长程稳定性。
  3. Qwen-AgentWorld — 复现 MCP、Terminal 或 SWE 环境评测，判断世界模型是否适合作为代理回归测试层。
  4. GPT-5.5 Instant — 用现有多轮约束提示词做回归测试，比较新版在意图理解、推荐和复杂约束遵循上的变化。
  5. GPT-5.6 Sol / Terra / Luna — 建立高难推理、日常工作和高吞吐任务三类评测矩阵，不要用一个默认模型覆盖所有场景。
  6. Claude Tag — 在低风险 Slack 频道试点，先限定可访问工具、数据源、预算和审计日志。
  7. Mistral OCR 4 — 用含表格、签名和复杂版式的真实文档测试结构化区块、边界框和置信度输出。
  8. Gemini computer use — 选一个浏览器或桌面重复流程，验证模型操作 UI 的成功率、权限隔离和人工确认节点。
  9. Codex 移动端 + DigitalOcean 插件 — 用长耗时编码任务测试移动通知、文件预览、行内审阅和云环境后台持续运行。
