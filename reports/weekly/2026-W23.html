6月1日 – 7日

本周趋势

  • 代理模型竞争：Qwen3.7-Plus 与 Grok Build Composer 2.5 同周更新，模型发布开始直接围绕浏览器操作和构建工作流展开。
  • 编码平台外扩：OpenAI Codex 走向角色插件、Sites 和 AWS，Claude Code 则用后台代理与 CLI 把开发工作流继续拆成可并行任务。
  • 科研助手成形：Google DeepMind 发布 Gemini for Science，把多智能体系统包装成假设生成、文献推理和实验设计的科研工作台。
  • 安全治理加码：Anthropic 扩大 Project Glasswing，OpenAI 推动青少年 AI 安全和 Foundation 韧性资助，治理议题继续机构化。
  • 语音进入流程：Travelers 的 OpenAI 实时语音理赔助手与 Voice Hack Night 原型显示，语音 Agent 正从演示走向企业和移动端流程。

Top 10 最重要文章

1. Qwen3.7-Plus 发布多模态代理模型 ⭐⭐⭐⭐⭐

  • 来源: Qwen (Twitter) | 分类: 模型发布 | 日期: 06/01
  • 标签: Qwen3.7-Plus 多模态代理 浏览器智能体 模型发布 工具调用

摘要: Qwen 发布 Qwen3.7-Plus，并把重点放在多模态代理和浏览器操作能力上。演示强调模型可以在网页环境中处理文本、视觉和工具调用混合任务，信号不只是参数升级，而是面向可执行流程的代理模型竞争。对国内和开源模型生态而言，这是把模型能力直接推向应用侧评测的一次重要发布。

2. Claude Code 同日升级：后台代理、CLI 与修复 ⭐⭐⭐⭐

  • 来源: Claude Devs (Twitter) | 分类: 产品与功能 | 日期: 06/02
  • 标签: Claude Code 后台代理 终端 CLI 工作流 Opus 4.8 子代理

摘要: Claude Code 同日推进多项开发者工作流更新：/fork 可把当前上下文拆给后台代理并行处理，CLI 则连接到托管智能体会话和 Claude API 技能。官方还说明并修复 Opus 4.8 工具调用与子代理耗额问题，让长期任务的成本和可靠性更容易检查。整体方向是把 Claude Code 从单会话助手继续推向可编排的开发平台。

3. Gemini for Science 发布多智能体科研助手 ⭐⭐⭐⭐

  • 来源: Google DeepMind (Twitter) | 分类: 研究 | 日期: 06/02
  • 标签: Gemini for Science 科研智能体 Co-Scientist 多智能体 假设生成

摘要: Google DeepMind 发布 Gemini for Science，把多智能体系统用于假设生成、文献推理和实验设计。演示中的 Co-Scientist 不只是回答科学问题，而是辅助提出候选方向并支持研究者试用。它代表 Gemini 能力被包装成面向科学发现的专用工作台。

4. Codex 扩展到角色插件、Sites 与知识工作 ⭐⭐⭐⭐

  • 来源: OpenAI Blog | 分类: 产品与功能 | 日期: 06/02
  • 标签: Codex 角色插件 Codex Sites 知识工作 团队协作 工作流

摘要: OpenAI 将 Codex 从编程助手扩展为多职能工作流平台，推出角色化插件、可分享的 Codex Sites 和团队批注能力。官方示例覆盖销售、金融研究和原型评审，强调非工程岗位也能把任务交给 Codex 生成可审阅产物。这延续了 Codex 从编码工具走向企业协作层的产品化路线。

5. Project Glasswing 扩至 150 个组织 ⭐⭐⭐⭐

  • 来源: Anthropic Blog | 分类: 政策与安全 | 日期: 06/02
  • 标签: Project Glasswing AI 安全 公共利益 社会韧性 Anthropic

摘要: Anthropic 扩大 Project Glasswing，把 AI 安全、教育和公共利益相关支持扩展到 150 个组织。该项目强调社会韧性与公益能力建设，体现 Anthropic 在产品和模型之外继续加码制度化影响力。与单点安全声明相比，这更像是一条长期合作网络的建设线。

6. OpenAI 模型与 Codex 在 AWS 全面可用 ⭐⭐⭐⭐

  • 来源: OpenAI (Twitter) | 分类: 产品与功能 | 日期: 06/01
  • 标签: OpenAI AWS Bedrock Codex 企业部署

摘要: OpenAI 宣布前沿模型与 Codex 进入 AWS 工作流并达到 GA，可通过 Bedrock 等 AWS 开发生态使用。这个动作降低了企业在既有 AWS 权限、账单和合规体系内试用 OpenAI 编程代理的门槛。对企业采购路径而言，Codex 正从独立开发工具进入标准云平台分发。

7. xAI 发布 Grok Build Composer 2.5 ⭐⭐⭐⭐

  • 来源: xAI (Twitter) | 分类: 模型发布 | 日期: 06/01
  • 标签: xAI Grok Build Composer 2.5 模型发布 开发工具

摘要: xAI 在 Grok Build 中上线 Composer 2.5，定位为面向构建与迭代的模型和工具能力升级。公开信息主要围绕入口和能力展示，用户可从 x.ai/cli 开始并在模型菜单中选择 Composer。它说明 xAI 也在把模型发布直接嵌入开发和原型构建场景。

8. Anthropic 支持美国 AI 行政令 ⭐⭐⭐

  • 来源: Anthropic (Twitter) | 分类: 政策与安全 | 日期: 06/02
  • 标签: 美国政策 AI 安全 行政令 Anthropic

摘要: Anthropic 表示，白宫关于促进先进人工智能创新与安全的行政令是强化美国 AI 领导力的重要一步。公司称将与白宫协作支持行政令实施。这条信息本身简短，但放在 Project Glasswing 和 OpenAI 青少年安全倡议旁边看，显示美国 AI 公司继续主动参与政策议程。

9. OpenAI 倡议建立国际青少年 AI 安全机制 ⭐⭐⭐

  • 来源: OpenAI Blog | 分类: 政策与安全 | 日期: 06/02
  • 标签: OpenAI 青少年安全 G7 AI 治理 家长控制

摘要: OpenAI 呼吁建立持续的国际青少年 AI 安全协作机制，并提出可由新机构或现有国家 AI 机构承担研究、证据共享和实践指导职责。文章列出九项原则，覆盖年龄识别、年度风险评估、家长控制、透明政策、严重事件处理、未成年人隐私和独立审计。OpenAI 将在 G7 相关讨论中推动该议题，把教育部署和青少年保护并列为治理目标。

10. OpenAI Foundation 投入 AI 韧性建设 ⭐⭐⭐

  • 来源: Sam Altman (Twitter) | 分类: 政策与安全 | 日期: 06/01
  • 标签: AI 韧性 OpenAI Foundation 模型安全 网络安全 资助

摘要: Sam Altman 转发 OpenAI Foundation 的 AI Resilience 愿景，强调社会管理 AI 风险的能力需要同步提升。相关计划已启动超过 1.3 亿美元初始资助，覆盖生物韧性、网络韧性、模型安全和 AI 对年轻人的影响。它与 OpenAI 的青少年安全倡议共同构成一条更宽的社会韧性路线。

分类概览

模型发布

本周模型发布数量不多，但主题非常集中：模型不再只强调聊天或基准，而是直接面向代理执行。Qwen3.7-Plus 主打多模态代理和浏览器操作能力，把视觉理解、网页状态和工具调用放到同一套流程中检验。xAI 的 Grok Build Composer 2.5 则嵌在构建工具内发布，强调开发者可以直接在 CLI 和模型菜单中试用。两条新闻共同说明，模型发布正在更紧地绑定具体工作台和操作环境。

研究

研究板块的核心是 Google DeepMind 的 Gemini for Science。它把 Gemini 包装成多智能体科研助手，覆盖假设生成、文献推理和实验设计，而不是停留在通用问答增强。Co-Scientist 的演示显示，AI 科研工具正在从“帮我解释论文”走向“帮我组织研究路径”。本周历史窗口里研究文章较少，因此这一条几乎独占了该分类的主线。

产品与功能

产品与功能继续由编码和工作流平台主导。OpenAI 把 Codex 扩展到角色插件、Codex Sites、团队批注和 AWS 分发，试图让非工程岗位也能使用 Codex 生成可审阅产物。Claude Code 则通过后台代理、CLI 和托管会话连接，把本地开发任务拆成更可并行的子流程。OpenAI Devs 的 Voice Hack Night 原型补上了语音 Agent 方向，说明移动端和语音壳层仍在寻找新的交互入口。

商业动态

商业动态本周主要体现为客户案例和行业落地。Travelers 使用 OpenAI Realtime API 和前沿模型构建车险理赔语音助手，两个月内从 8 个州扩展到全美，且使用该助手的客户中有 85% 到 90% 完成理赔提交。Claude 展示 Legora 法律工作流案例，强调专业服务中的文档、检索和协作流程。两者都不是平台级发布，但说明 AI 落地正在进入保险和法律这类流程重、合规重的行业。

政策与安全

政策与安全是本周最密集的分类之一。Anthropic 扩大 Project Glasswing 至 150 个组织，并公开支持美国 AI 行政令，显示其政策参与方式从表态走向合作网络建设。OpenAI 则同时推动国际青少年 AI 安全机制和 Foundation 韧性资助，覆盖年龄识别、家长控制、独立审计、生物韧性、网络韧性和模型安全。整体看，头部公司正在把安全治理拆成可资助、可审计、可外交协调的制度化项目。

教程与观点

教程与观点较轻量，主要是提示词和使用观察。Thariq 转述 Anthropic 员工关于 Claude 跟进提示的方法，强调一次回答之后继续追问约束、改写和验证，适合沉淀为团队提示词手册。另有语音模式用于更自然回复的观察，以及 Sam Altman 分享个人座右铭。该分类本周缺少高影响长文，但补充了日常使用层面的细碎经验。

本周值得上手试试

  1. Qwen3.7-Plus 浏览器代理 — 用表单填写、网页检索和多步骤后台操作做一组同题测试，重点记录视觉理解、工具调用失败和页面状态恢复能力。
  2. Claude Code /fork 后台代理 — 在当前开发上下文中用 /fork 拆出一个独立子任务，观察后台代理是否能减少等待和上下文切换。
  3. Codex 角色插件与 Sites — 为销售、研究或产品运营各设计一个小型 Codex 插件原型，再用 Codex Sites 生成可批注的共享页面。
  4. Codex on AWS / Bedrock — 检查现有 AWS 账户能否访问 OpenAI 模型和 Codex，用一个小型集成验证权限、审计、账单和延迟差异。
  5. Grok Build Composer 2.5 — 从 x.ai/cli 进入 Grok Build，在同一个原型任务上对比 Composer 2.5 与现有代码/界面生成工具的多轮修改质量。
  6. 语音 Agent 原型 — 选一个手机端高频任务，把点击流程改写成语音优先流程，参考 Voice Hack Night 项目验证指令理解和动作确认体验。

值得深读

  • Qwen3.7-Plus 发布多模态代理模型 — 它把多模态、浏览器操作和工具调用放在同一条代理模型主线上，是观察国产模型应用侧竞争的重要入口。
  • Codex 扩展到角色插件、Sites 与知识工作 — 这篇文章说明 OpenAI 正把 Codex 从工程工具扩展成面向多岗位的协作和产物生成平台。
  • Gemini for Science 发布多智能体科研助手 — DeepMind 把 Gemini 产品化为科研工作台，值得关注其对假设生成和实验设计流程的影响。
  • Project Glasswing 扩至 150 个组织 — Anthropic 的安全与公益网络建设，比单次政策表态更能体现头部 AI 公司的长期治理策略。
  • Travelers 用 OpenAI 扩展理赔助手 — 保险理赔语音助手的完成率和扩展速度，是 Realtime API 进入复杂企业流程的一个清晰案例。
