6月1日 – 7日

本周趋势

  • 代理模型落地：Qwen3.7-Plus 与 Grok Build Composer 2.5 把模型更新直接绑定到浏览器操作、工具调用和原型构建场景。

Top 10 最重要文章

1. Qwen3.7-Plus 发布多模态代理模型 ⭐⭐⭐⭐⭐

  • 来源: Qwen (Twitter) | 分类: 模型发布 | 日期: 06/01
  • 标签: Qwen3.7-Plus 多模态代理 浏览器智能体 模型发布 工具调用

摘要: Qwen 发布 Qwen3.7-Plus，并把演示重点放在多模态代理、浏览器操作和工具调用上。这个版本的信号不只是新模型编号，而是把文本、视觉、网页状态和动作规划放进同一条可执行任务链。对国产和开源模型生态来说，浏览器智能体能力会成为更接近真实应用的对照指标。

2. Claude Code 同日升级：后台代理、CLI 与修复 ⭐⭐⭐⭐

  • 来源: Claude Devs (Twitter) | 分类: 产品与功能 | 日期: 06/02
  • 标签: Claude Code 后台代理 终端 CLI 工作流 Opus 4.8 子代理

摘要: Claude Code 通过 /fork 增加后台代理能力，让开发者可以把当前上下文拆出去并行处理独立子任务。Claude Platform 还把终端 CLI、托管智能体会话和 Claude API 技能串联起来，减少本地开发与平台功能之间的切换成本。官方同步解释并修复 Opus 4.8 工具调用和子代理耗额问题，说明 Anthropic 正在补齐长任务编排里的可靠性细节。

3. Gemini for Science 发布多智能体科研助手 ⭐⭐⭐⭐

  • 来源: Google DeepMind (Twitter) | 分类: 研究 | 日期: 06/02
  • 标签: Gemini for Science 科研智能体 Co-Scientist 多智能体 假设生成

摘要: Google DeepMind 发布 Gemini for Science，把 Gemini 包装成面向科学发现的多智能体研究助手。Co-Scientist 演示覆盖假设生成、文献推理和实验设计，重点是帮助研究者组织候选方向，而不是单纯解释论文。它代表通用模型能力正在被装进更专用的科研工作台。

4. Codex 扩展到角色插件、Sites 与知识工作 ⭐⭐⭐⭐

  • 来源: OpenAI Blog | 分类: 产品与功能 | 日期: 06/02
  • 标签: Codex 角色插件 Codex Sites 知识工作 团队协作 工作流

摘要: OpenAI 将 Codex 从编程助手继续扩展为多职能工作流平台，发布角色插件、Codex Sites 和团队批注能力。官方示例覆盖销售、金融研究和原型评审，目标是让非工程岗位也能把任务交给 Codex 生成可审阅产物。Codex 的产品边界因此从“写代码”扩到“把知识工作变成可共享工作流”。

5. Project Glasswing 扩至 150 个组织 ⭐⭐⭐⭐

  • 来源: Anthropic Blog | 分类: 政策与安全 | 日期: 06/02
  • 标签: Project Glasswing AI 安全 公共利益 社会韧性 Anthropic

摘要: Anthropic 扩大 Project Glasswing，将 AI 安全、教育与公共利益相关支持扩展到 150 个组织。相比单次政策表态，这更像是面向社会韧性和公益能力建设的长期合作网络。它也延续了 Anthropic 把安全治理放进组织、课程和伙伴关系中的路线。

6. OpenAI 模型与 Codex 在 AWS 全面可用 ⭐⭐⭐⭐

  • 来源: OpenAI (Twitter) | 分类: 产品与功能 | 日期: 06/01
  • 标签: OpenAI AWS Bedrock Codex 企业部署

摘要: OpenAI 宣布前沿模型与 Codex 在 AWS 工作流中 GA，可通过 Bedrock 等 AWS 开发生态使用。这个分发路径让企业可以在既有权限、审计、账单和合规体系中试用 OpenAI 编程代理。对 Codex 来说，进入 AWS 意味着它正在从单独工具变成企业云采购路径里的标准能力。

7. xAI 发布 Grok Build Composer 2.5 ⭐⭐⭐⭐

  • 来源: xAI (Twitter) | 分类: 模型发布 | 日期: 06/01
  • 标签: xAI Grok Build Composer 2.5 模型发布 开发工具

摘要: xAI 在 Grok Build 中上线 Composer 2.5，定位为面向构建和迭代的模型能力升级。公开信息主要提供试用入口和能力展示，用户可从 x.ai/cli 开始并在模型菜单中选择 Composer。虽然技术细节有限，但它表明 xAI 也在把模型更新直接嵌入开发工具链。

8. Anthropic 支持美国 AI 行政令 ⭐⭐⭐

  • 来源: Anthropic (Twitter) | 分类: 政策与安全 | 日期: 06/02
  • 标签: 美国政策 AI 安全 行政令 Anthropic

摘要: Anthropic 表态支持白宫关于促进先进人工智能创新与安全的行政令，并称会协作推进实施。单条信息较短，但与 Project Glasswing 扩张放在一起看，说明 Anthropic 正同时经营政策响应和社会安全网络。头部 AI 公司已经越来越主动地进入国家 AI 治理议程。

9. OpenAI 倡议建立国际青少年 AI 安全机制 ⭐⭐⭐

  • 来源: OpenAI Blog | 分类: 政策与安全 | 日期: 06/02
  • 标签: OpenAI 青少年安全 G7 AI 治理 家长控制

摘要: OpenAI 呼吁围绕青少年 AI 安全建立持续国际协作机制，并建议由新机构或现有国家 AI 机构承担研究、证据共享和实践指导职能。文章提出九项原则，覆盖隐私保护的年龄识别、年度风险评估、家长控制、透明政策、严重事件处理、未成年人隐私和独立审计。OpenAI 计划在 G7 相关讨论中推动该议题，把教育机会和青少年保护作为同一治理框架的两面。

10. OpenAI Foundation 投入 AI 韧性建设 ⭐⭐⭐

  • 来源: Sam Altman (Twitter) | 分类: 政策与安全 | 日期: 06/01
  • 标签: AI 韧性 OpenAI Foundation 模型安全 网络安全 资助

摘要: Sam Altman 转发 OpenAI Foundation 的 AI Resilience 愿景，强调社会管理 AI 风险的能力需要与模型能力同步提升。该计划已启动超过 1.3 亿美元初始资助，覆盖生物韧性、网络韧性、模型安全和 AI 对年轻人的影响。它与青少年安全倡议共同说明 OpenAI 正把风险治理拆成可资助、可研究、可协作的项目组合。

分类概览

模型发布

模型发布本周只有两条，但都指向同一个方向：模型更新正在与具体代理环境绑定。Qwen3.7-Plus 强调多模态代理和浏览器操作，把网页理解、动作规划和工具调用放到实际任务里检验。xAI 的 Grok Build Composer 2.5 则嵌在 Grok Build 和 CLI 使用路径里，面向原型构建与多轮修改。模型竞争的比较对象因此不只是跑分，还包括能否稳定完成复杂界面和工具链任务。

研究

研究板块主要由 Google DeepMind 的 Gemini for Science 支撑。它把 Gemini 放进科研流程，面向假设生成、文献推理和实验设计组织多智能体协作。Co-Scientist 的价值在于把模型从问答助手推进到研究路径生成器，让研究者可以围绕候选方向继续验证。由于本周历史窗口里的研究条目较少，这条新闻构成了该分类的主要观察点。

产品与功能

产品与功能仍然围绕“把 Agent 做成平台”展开。OpenAI 的 Codex 更新覆盖角色插件、Codex Sites、团队批注和 AWS 分发，把工程能力延伸到销售、金融研究、产品评审等知识工作场景。Anthropic 的 Claude Code 则通过后台代理、CLI 和托管会话连接，把开发任务进一步拆成可并行、可追踪的工作流。OpenAI Devs 的 Voice Hack Night 原型补充了语音优先移动 Agent 的方向，说明入口形态仍在快速试探。

商业动态

商业动态更多体现为企业案例，而不是融资或并购。Travelers 基于 OpenAI Realtime API 和前沿模型构建车险理赔语音助手，两个月内从 8 个州扩展到全美，使用该助手的客户中有 85% 到 90% 完成理赔提交。Claude 官方展示 Legora 法律工作流案例，突出 AI 在专业服务里的文档、检索和协作价值。这两条新闻共同说明，语音和文档型 Agent 正进入流程复杂、合规要求高的行业。

政策与安全

政策与安全本周非常集中，核心是头部公司把治理议题项目化。Anthropic 将 Project Glasswing 扩大至 150 个组织，并公开支持美国 AI 行政令，形成“合作网络 + 政策响应”的组合。OpenAI 则提出国际青少年 AI 安全机制，并通过 Foundation 投入 AI 韧性建设，覆盖生物、网络、模型安全和未成年人影响。安全治理不再只是原则声明，而是变成资助、审计、机构协作和外交讨论的具体议程。

教程与观点

教程与观点板块偏使用经验。Thariq 转述 Anthropic 员工关于 Claude 跟进提示的方法，强调拿到初稿后继续追问约束、改写和验证，而不是把一次回答当成终稿。另有语音模式用于更自然回复的观察，以及 Sam Altman 的个人引文分享。整体信息量不高，但适合把“后续追问模板”和“语音交互观察”沉淀进团队使用手册。

本周值得上手试试

  1. Qwen3.7-Plus 浏览器代理 — 用网页检索、表单填写和多步骤后台操作做同题测试，记录视觉理解、工具调用、页面状态恢复和长任务稳定性。
  2. Claude Code 后台代理 — 在一个真实开发任务里用 /fork 拆出独立子任务，比较后台代理对等待时间、上下文切换和子任务完成率的影响。
  3. Claude CLI 工作流 — 把 Claude CLI、托管会话和 API 技能接入本地脚本，验证是否能减少从终端跳到网页或 IDE 的操作成本。
  4. Codex 角色插件 — 为销售、金融研究或产品运营各设计一个高频任务插件，用同一输入生成可审阅产物并收集团队批注。
  5. Codex Sites — 用 Codex Sites 发布一个可分享原型，让业务同事直接在页面上批注，再评估修改回路是否比文档评审更快。
  6. Codex on AWS — 在现有 AWS 账户里验证 OpenAI 模型与 Codex 的可用性，重点检查权限、审计、账单、延迟和直接 API 调用的差异。
  7. Grok Build Composer 2.5 — 从 x.ai/cli 启动一个已有原型任务，对比 Composer 2.5 在代码生成、界面修改和多轮迭代中的失败模式。
  8. 语音 Agent 原型 — 选择一个手机端高频任务，把点击路径改写成语音优先流程，参考 Voice Hack Night 项目验证确认机制和误触恢复。

值得深读

  • Codex 扩展到角色插件、Sites 与知识工作 — 这篇文章展示 OpenAI 如何把 Codex 从工程助手扩成多岗位工作流平台。

  • Project Glasswing 扩至 150 个组织 — Anthropic 的项目扩张体现了 AI 安全治理从声明走向合作网络和公益能力建设。
