6月1日 – 7日

本周趋势

  • 垂直模型升温：OpenAI GPT-Rosalind 与 Anthropic Claude 化学研究把 GPT-5.5、Opus 4.7 推进生命科学和 NMR 专业任务。
  • 本地多模态补强：Google Gemma 4 12B 与 Qwen3.7-Plus 分别强化端侧多模态和浏览器代理能力，模型竞争继续贴近执行场景。
  • 编码工作台扩展：Codex 角色插件、iOS 插件、设置恢复与 Claude CLI、/fork、ultracode 让编码代理更像长期可编排平台。
  • 安全治理转实操：Anthropic MITRE 映射、Project Glasswing 与 OpenAI 生物防御方案把 AI 风险分析推进到框架、韧性和防御能力。
  • 记忆与个性化回归：ChatGPT Memory 新系统用 summary 和用户控制增强跨会话上下文，个人长期助手体验成为产品焦点。
  • 企业分发加深：OpenAI 模型进入 AWS，Grok 接入 Cloudflare/Vapi/Gopuff，Claude Partner Network 扩容，模型能力继续通过平台外溢。

Top 10 最重要文章

1. Qwen3.7-Plus 发布多模态代理模型 ⭐⭐⭐⭐⭐

  • 来源: Qwen (Twitter) | 分类: 模型发布 | 日期: 06/01
  • 标签: Qwen3.7-Plus 多模态代理 浏览器智能体 模型发布 工具调用

摘要: Qwen 发布 Qwen3.7-Plus，并把重点放在多模态代理和浏览器操作能力上。演示强调模型可以在浏览器环境中执行任务，同时处理文本、视觉与工具混合流程。它延续了 Qwen3.7-Max 的 Agent 路线，把国产模型竞争从榜单推进到更贴近真实网页任务的能力验证。

2. GPT-Rosalind 升级：GPT-5.5 进入生命科学 ⭐⭐⭐⭐⭐

  • 来源: OpenAI Blog | 分类: 模型发布 | 日期: 06/03
  • 标签: GPT-Rosalind GPT-5.5 生命科学 药物发现 企业AI

摘要: OpenAI 为企业生命科学研究模型 GPT-Rosalind 推出新能力，把 GPT-5.5 的智能体编码、工具调用和专业推理用于药物化学、基因组学与实验工作流。更新后的模型在复杂药物化学查询、定量生物学和湿实验排错中表现更强，并以 trusted-access 结构向合格组织开放研究预览。这是 GPT-5.5 从通用编码/文档能力进入高价值垂直科学场景的重要一步。

3. Gemma 4 12B 发布：本地多模态模型升级 ⭐⭐⭐⭐⭐

  • 来源: Demis Hassabis (Twitter) | 分类: 模型发布 | 日期: 06/03
  • 标签: Gemma 4 12B模型 多模态 本地部署 Apache 2.0

摘要: Google 庆祝 Gemma 下载量突破 1.5 亿，并推出新的 Gemma 4 12B 模型。该模型采用统一、无编码器的多模态设计，面向本地笔记本运行场景，官方强调 16GB VRAM 设备也能承载，并以 Apache 2.0 许可证发布。它让端侧多模态从“能跑”进一步接近“可被企业和个人开发者认真评测”。

4. Claude Opus 4.7 挑战 NMR 化学解析 ⭐⭐⭐⭐⭐

  • 来源: Anthropic Research | 分类: 研究 | 日期: 06/05
  • 标签: Claude Opus 4.7 NMR 化学 科学AI Anthropic Research

摘要: Anthropic Research 发布 Making Claude a chemist，研究 Claude 在 NMR 光谱和化学结构理解任务中的能力。官方称 Opus 4.7 在部分任务上达到甚至超过专用 NMR 软件，显示旗舰模型正在进入高度专业的科学分析场景。与 GPT-Rosalind 同周出现，说明前沿模型的科学落地正在从泛科研助手走向具体学科工具。

5. xAI 发布 Grok Build Composer 2.5 ⭐⭐⭐⭐

  • 来源: xAI (Twitter) | 分类: 模型发布 | 日期: 06/01
  • 标签: xAI Grok Build Composer 2.5 模型发布 开发工具

摘要: xAI 在 Grok Build 中上线 Composer 2.5，定位为面向构建与迭代的模型和工具能力升级。公开信息主要围绕试用入口和能力展示，用户可从 x.ai/cli 开始并在模型菜单中选择 Composer。它说明 xAI 仍在把模型能力直接嵌入原型构建和开发工具链。

6. OpenAI 模型与 Codex 在 AWS 全面可用 ⭐⭐⭐⭐

  • 来源: OpenAI (Twitter) | 分类: 产品与功能 | 日期: 06/01
  • 标签: OpenAI AWS Bedrock Codex 企业部署

摘要: OpenAI 宣布前沿模型与 Codex 进入 AWS 工作流并达到 GA，可通过 Bedrock 等 AWS 开发生态使用。开发者信息补充了 Codex on Bedrock 的文档入口和集成方式，显示 OpenAI 正把模型与编码代理放进更多企业云采购路径。对团队而言，这降低了在既有 AWS 权限、账单和合规体系内试用 Codex 的门槛。

7. Project Glasswing 扩至 150 个组织 ⭐⭐⭐⭐

  • 来源: Anthropic Blog | 分类: 政策与安全 | 日期: 06/02
  • 标签: Project Glasswing AI 安全 公共利益 社会韧性 Anthropic

摘要: Anthropic 扩大 Project Glasswing，把 AI 安全、教育和公共利益相关支持扩展到 150 个组织。官方强调该项目面向社会韧性和公益能力建设，而不是单次产品功能。它与后续 MITRE 网络威胁报告共同说明 Anthropic 正把安全工作从模型评测推进到组织合作和行业框架。

8. Codex 扩展到角色插件、Sites 与知识工作 ⭐⭐⭐⭐

  • 来源: OpenAI Blog | 分类: 产品与功能 | 日期: 06/02
  • 标签: Codex 角色插件 Codex Sites 知识工作 团队协作 工作流

摘要: OpenAI 将 Codex 从编程助手扩展为多职能工作流平台，发布角色插件、可分享的 Codex Sites 和团队批注能力。官方示例覆盖销售、金融研究和原型评审，强调非工程岗位也能把任务拆给 Codex 生成可操作产物。同周 OpenAI 还补充称角色插件覆盖 62 个应用与 110 个技能，Codex 的定位正在从编码工具扩展到团队工作台。

9. Gemini for Science 发布多智能体科研助手 ⭐⭐⭐⭐

  • 来源: Google DeepMind (Twitter) | 分类: 研究 | 日期: 06/02
  • 标签: Gemini for Science 科研智能体 Co-Scientist 多智能体 假设生成

摘要: Google DeepMind 发布 Gemini for Science，将多智能体系统用于假设生成、文献推理和实验设计等科研流程。演示中的 Co-Scientist 不只是回答科学问题，而是帮助研究者提出候选方向并组织验证路径。它与本周生命科学和化学模型进展呼应，显示科学 AI 正在从通用助手走向专门工作台。

10. Claude Platform CLI 打通 API 与 Managed Agents ⭐⭐⭐⭐

  • 来源: Claude Devs (Twitter) | 分类: 产品与功能 | 日期: 06/02
  • 标签: Claude Platform CLI Managed Agents Claude Code API

摘要: Claude Devs 发布 Claude Platform CLI，让每个 API endpoint 都可从终端运行，包括 Messages API、Claude Managed Agents 和结果管道化。CLI 可启动 Managed Agents 会话、发送事件、查看完整 trace，并可被 Claude Code 通过内置 /claude-api skill 调用。它把平台操作纳入编码代理工作流，降低了调试、审计和自动化脚本的摩擦。

分类概览

模型发布

W23 的模型发布主线从通用旗舰转向“场景化模型”。Qwen3.7-Plus 主打多模态代理和浏览器操作，xAI 的 Composer 2.5 继续嵌在 Grok Build 构建工具里，说明模型发布越来越需要绑定可执行环境。OpenAI 的 GPT-Rosalind 把 GPT-5.5 带入生命科学，覆盖药物化学、基因组学和实验排错。Google 的 Gemma 4 12B 则补强本地多模态路线，以 16GB VRAM 和 Apache 2.0 许可降低端侧部署门槛。

研究

研究板块的重心是科学能力和自我改进风险。Anthropic 的 Claude 化学/NMR 研究显示 Opus 4.7 已能在部分专业任务上达到或超过专用软件，和 GPT-Rosalind 形成“生命科学 + 化学分析”的双线信号。Google DeepMind 的 Gemini for Science 把多智能体系统用于假设生成和实验设计。Anthropic 还披露 Claude 正在加速 AI 研发自我改进，提示递归自我改进虽未确定到来，但研究流程加速本身已经值得持续监测。

产品与功能

产品与功能依然由 Codex、Claude Code 和平台分发主导。OpenAI 把 Codex 推进角色插件、Sites、AWS、iOS 插件、设置搜索和状态保持，让它覆盖非工程知识工作、企业云和移动/iOS 开发循环。Anthropic 则通过 Claude Platform CLI、/fork 后台代理、ultracode 动态工作流和自检流程，把 Claude Code 推向可编排、可审计的长期任务平台。ChatGPT Memory 升级是另一条重要产品线，它把跨会话上下文、summary 和用户控制放回个人助手体验中心。

商业动态

商业动态数量不高，但方向清晰：模型能力继续通过行业与伙伴网络扩散。Travelers 用 OpenAI Realtime API 构建车险理赔语音助手，两个月内从 8 个州扩展到全美，显示实时语音进入复杂企业流程。Anthropic 扩展 Claude Partner Network，新增 Services Track 和 Partner Hub，目标是把 Claude 从试点推到可运营生产系统。Berkeley RDI 的 Agentic AI Summit 与 Claude/Legora 法律工作流案例则补充了生态活动和专业服务落地。

政策与安全

政策与安全本周明显从原则声明转向可操作框架。Anthropic 将 832 个恶意账号的 AI 网络威胁行为映射到 MITRE ATT&CK，说明安全社区既有技战术语言需要适配 AI 辅助攻击。OpenAI 提出生物防御智能时代行动方案，把生命科学 AI 的进步和滥用风险放在同一治理框架下。Project Glasswing 扩容、OpenAI 青少年安全倡议、OpenAI Foundation 韧性资助和美国 AI 行政令回应共同构成一条社会韧性与国家治理线。

教程与观点

教程与观点主要围绕“如何把 Agent 用进真实工作”。Wasmer 案例展示 Codex 和 GPT-5.5 如何把边缘 Node.js/WebAssembly 工程从一年级别压缩到两周，Endava 则把 ChatGPT Enterprise、Codex 和 AI-native 交付文化结合起来。Claude Devs 分享商业分析智能体实践，强调 skills、数据基础和评测。Thariq 的 Claude 跟进提示法、个人软件复兴观点，以及 Claude/Lovable 的信任护城河讨论，说明工具使用方法正在从 prompt 技巧扩展到产品和组织形态。

本周值得上手试试

  1. Qwen3.7-Plus 浏览器代理 — 用表单填写、网页检索和多步骤后台操作做同题测试，重点记录视觉理解、工具调用、页面状态恢复和长任务稳定性。
  2. GPT-Rosalind 生命科学评测 — 整理药物化学、基因组学和实验排错任务集，把工具调用、代码生成和领域专家审阅放进同一评测流。
  3. Gemma 4 12B 本地多模态 — 在 16GB VRAM 设备上测试真实图文推理、端侧延迟和 Apache 2.0 许可下的内部部署可行性。
  4. Codex on AWS — 用 Bedrock 路径做一个小型 Codex 集成验证，重点观察权限、审计、账单、延迟和直接 API 调用差异。
  5. Codex 角色插件 + Sites — 选择销售、数据分析或产品原型流程，安装角色插件并用 Sites 生成可评审输出，提前定义分享范围和验收标准。
  6. Claude Platform CLI — 用 CLI 封装 API 调试、文件上传、Managed Agents 事件脚本和 trace 拉取，纳入 Claude Code 工作流。
  7. Claude Code /fork 与 ultracode — 用 /fork 跑高风险改动的后台候选方案，并把常见复杂任务迁移到 ultracode 动态工作流。
  8. ChatGPT Memory — 在长期项目对话中测试 memory summary 是否准确保留偏好、约束和项目状态，并建立敏感记忆清理流程。
  9. Codex iOS 插件 — 用一个 SwiftUI 小项目验证预览、模拟器、截图预览、热更新和失败日志是否能稳定进入 Codex 自检循环。
  10. OpenAI moderation signals — 在 Responses/Completions 请求中同时返回 moderation 分数，先用于日志和人工复核，再定义阻断阈值。

值得深读

  • GPT-Rosalind 升级：GPT-5.5 进入生命科学 — 它是 GPT-5.5 垂直化落地的代表，值得看模型如何进入药物化学、基因组学和湿实验流程。
  • Claude Opus 4.7 挑战 NMR 化学解析 — Anthropic 把 Claude 放进 NMR 化学结构解析，是理解科学 AI 专业化边界的重要案例。
  • Codex 扩展到角色插件、Sites 与知识工作 — 这篇文章展示 Codex 如何从编码助手扩成多岗位工作流平台。
  • Anthropic 用 MITRE 映射 AI 网络威胁 — 它把 AI 滥用行为映射到 MITRE ATT&CK，适合安全团队评估现有框架是否足够。
  • ChatGPT Memory 升级：上下文跨会话保鲜 — Memory 的 summary、控制和新鲜度机制决定长期个人助手体验是否真正可用。
