1. The next evolution of the Agents SDK

  • 来源: OpenAI Blog
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: OpenAI Agents SDK 沙箱 智能体 开发者工具

摘要: OpenAI 宣布 Agents SDK 的下一代演进，引入原生沙箱执行能力与模型原生 harness，帮助开发者在跨文件与工具场景中构建安全、可长时运行的 AI 智能体。此次更新是 OpenAI 持续加固企业级智能体基础设施、降低开发者自行搭建安全运行时负担的关键一步。

实践建议

  • 试用新版 Agents SDK 的 native sandbox 模式，替换自建的进程隔离方案，测试复杂工具调用链的稳定性
  • 把长时运行的智能体任务从自建容器迁移到 SDK 的 model-native harness，对比运维成本与失败恢复表现

2. Gemini 3.1 Flash TTS：最具可控性的 AI 语音模型

  • 来源: Google AI Blog | 📡 也被 Google DeepMind (Twitter) 报道
  • 分类: 模型发布
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Gemini TTS Audio Tags 多语言 SynthID

摘要: Google 发布 Gemini 3.1 Flash TTS，主打"可控性"——通过全新的 Audio Tags，可用自然语言指令直接调节语气、语速与风格；支持 70+ 种语言（含印地语、日语、德语），所有输出均带 SynthID 水印。该模型已通过 Gemini API / Google AI Studio 面向开发者开放预览，企业通过 Vertex AI 滚动上线，消费者侧则率先接入 Google Vids。

实践建议

  • 在 Google AI Studio 用 Audio Tags 对同一段脚本调节情感/节奏，评估相比 ElevenLabs、OpenAI TTS 的可控性差异
  • 将 SynthID 水印鉴别纳入内容合规链路，测试生成语音在下游二次剪辑后的水印可追溯能力

3. Anthropic 等合著"潜意识学习"研究登上 Nature

  • 来源: Anthropic (Twitter)
  • 分类: 研究
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Anthropic 潜意识学习 Nature 对齐 合成数据

摘要: Anthropic 联合发表的 "Subliminal Learning" 研究正式被 Nature 收录。论文证明大语言模型能够通过看似与目标特征无关的数据（例如没有明显含义的数字序列）将偏好、甚至对齐缺陷暗中传递给学生模型，对合成数据训练与模型蒸馏的安全假设提出严峻挑战。该工作由 Owain Evans 团队领衔，Anthropic 与多方合作完成。

4. 使用 Claude Code：会话管理与 100 万上下文

  • 来源: 宝玉的分享
  • 分类: 教程与观点
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: Claude Code 上下文管理 子智能体 Thariq 最佳实践

摘要: 宝玉翻译整理了 Claude Code 核心布道者 Thariq 的深度经验：何时开新会话、回溯（rewind）与纠正（correction）的取舍、上下文压缩（compact）与清空（clear）的时机、子智能体何时真正有用，以及如何规避糟糕的上下文压缩。面向已在日常使用 Claude Code 的开发者，是进入百万上下文时代的一份操作手册。

5. Qwen3.6-Plus 与 3.5-Plus 正式登陆 OpenCode

  • 来源: Qwen (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: Qwen OpenCode 编码助手 多模态 零数据保留

摘要: Qwen 宣布 Qwen3.6-Plus 与 Qwen3.5-Plus 已集成进开源编码助手 OpenCode，两者均支持图像输入与零数据保留，其中 Qwen3.5-Plus 定价约为 3.6-Plus 的 1/3，主打成本敏感场景。此次集成把 Qwen 模型更深地嵌入开发者的命令行/IDE 日常工作流。

实践建议

  • 在 OpenCode 里切换 Qwen3.5-Plus 与 Qwen3.6-Plus 跑同一套真实编码任务（重构、补测试、代码 review），评估 3x 价格差下的质量差异
  • 为使用 Qwen 的团队接入 3.6-Plus 的图像能力，测试截图驱动的 UI 缺陷修复或多模态 PR 审查
