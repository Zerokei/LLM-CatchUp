3. ChatGPT Memory 升级：上下文跨会话保鲜

  • 来源: OpenAI Blog
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: ChatGPT Memory 个性化 上下文 用户控制

摘要: OpenAI 推出新的 ChatGPT memory 系统，让 ChatGPT 能更好地跨对话保留偏好和重要上下文，并保持记忆的新鲜度与相关性。用户可以通过 memory summary 查看和引导系统记住什么，也可以回到旧版 saved memories 体验；该能力先向美国 Plus 和 Pro 用户推出。Sam Altman 与 OpenAI 社媒均将其描述为一次明显的记忆能力升级。

📎 多角度报道:

  • ChatGPT Memory 增加摘要与可控性 · OpenAI (Twitter) · 控制细节
  • Altman 强调 ChatGPT Memory 大升级 · Sam Altman (Twitter) · Altman 背书
  • OpenAI 社媒发布 ChatGPT Memory 升级 · OpenAI (Twitter) · 社媒公告

实践建议

  • 在长期项目对话中测试 memory summary 是否能准确保留偏好、约束和项目状态。
  • 为敏感任务建立定期检查与清理 memory 的流程，避免过期上下文影响结果。
  • 比较新版 memory 与旧版 saved memories 在连续工作流中的召回质量。

7. OpenAI API 返回 Moderation Scores

  • 来源: OpenAI Devs (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: Responses API Completions API Moderation 安全路由

摘要: OpenAI Devs 宣布 Responses API 和 Completions API 可在同一请求流中返回 moderation signals。开发者可以把这些分数用于日志、路由、人工复核或阻断，从而把安全判断嵌入生成链路。

实践建议

  • 在生成请求中同时返回 moderation signals，并定义日志、复核、阻断的阈值策略。
  • 把 moderation 分数纳入灰度路由，先观察误报和漏报。
