1. Codex 扩展：移动端全量 + 云端插件

  • 来源: OpenAI Devs (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Codex 移动端 DigitalOcean 云开发环境 代码审阅 多智能体

摘要: Codex 在 ChatGPT 移动应用中正式全量开放，加入一对一设备配对、通知、目标、侧边聊天、文件预览和行内审阅评论，让长耗时编码任务更容易在手机上跟进。同一天，DigitalOcean 插件让用户用一句提示创建运行在自己云账户中的持久化开发环境，Codex 可在用户离开后继续工作。OpenAI Devs 还补充了长线程、连接和 diff 处理改进，并通过 Builders Unscripted 展示 GPT-5.5 与多 agent Codex 工作流的创意软件实践。 OpenAI Devs 表示 Codex 移动端修复了多项体验问题，包括更好处理长线程、更可靠的连接以及改进 diff 处理。该更新偏向稳定性和开发者体验优化，而非新能力发布。

📎 多角度报道:

  • DigitalOcean 插件接入 Codex 云开发 · OpenAI Devs (Twitter) · 云开发插件
  • OpenAI Devs 分享创意 Codex 工作流 · OpenAI Devs (Twitter) · 创意工作流访谈

实践建议

  • 用长耗时编码任务测试移动端通知、文件预览和行内审阅流程。
  • 试跑 DigitalOcean 插件，验证环境创建、依赖安装和后台持续运行。
  • 检查云账户权限、费用上限和闲置环境回收策略。
  • 把移动端配对纳入团队设备安全规范。

2. Claude Tag 展示多人主动智能体

  • 来源: Claude Devs (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Claude Tag Claude Code 多人智能体 长期记忆 协作工作流 主动代理

摘要: Claude Tag 被定位为建立在 Claude Code 之上的主动式多人智能体，核心特征是记忆、身份和持续协作。官方深度解析说明它如何从单次编码助手走向可参与团队流程的代理形态，Thariq 的补充则把它放到后续访谈和视频演示场景中。这个发布对 Claude Code 用户的意义在于，团队协作、长期上下文和 agent 身份管理正在成为产品层能力。

📎 多角度报道:

  • Thariq 预告 Claude Tag 深聊 · Thariq (Twitter) · 访谈与演示补充

实践建议

  • 阅读官方 deep dive，提炼记忆、身份和权限边界的设计模式。
  • 选择一个跨人协作开发任务，验证主动式 agent 是否减少交接成本。
  • 把现有 Claude Code 工作流区分为单人任务和多人任务，评估迁移优先级。

3. Gemini Flash 支持原生电脑操作

  • 来源: Google DeepMind (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Gemini Computer Use 智能体 自动化 多端交互

摘要: Google DeepMind 宣布 Gemini 3.5 Flash 支持原生 computer use，开发者可构建能观察并操作浏览器、移动端和桌面界面的自定义智能体。这把界面操作能力直接纳入模型工具链，对自动化测试、办公流程和跨应用代理开发都有现实意义。

实践建议

  • 选取一个浏览器或桌面重复流程，验证 Gemini 原生 computer use 的可靠性边界。
  • 为代理操作增加权限隔离、审计日志和人工确认节点。
  • 对比现有 Playwright 或 RPA 方案，评估模型驱动操作的成本和成功率。

4. Anthropic 经济指数聚焦使用节奏

  • 来源: Anthropic Research
  • 分类: 研究
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: Anthropic Economic Index Claude 劳动力影响 使用行为

摘要: Anthropic 更新 Economic Index 方法，提升采样频率、引入输出分类器，并按 Claude 对话与 1P API 提供更细粒度数据。报告显示 Claude 使用会随工作周、日内节奏和税务截止日期波动，输出类型与产品形态强相关，而更高算力通常对应更高价值的产出。调查还发现，更自动化使用 Claude 的人更预期 AI 会承担更多任务，同时对薪酬、工作安全和工作意义更乐观。

5. OpenAI 披露 Codex 重塑内部工作

  • 来源: OpenAI (Twitter)
  • 分类: 研究
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: Codex 智能体工作 OpenAI 非开发者采用 知识工作

摘要: OpenAI 将 agentic AI 描述为把知识工作从短对话转向可委托的长周期任务，并以 Codex 在公司内部的采用作为早期样本。文章称 Codex 已成为 OpenAI 各部门的主要 AI 工具，非开发者使用增长尤其快，个人用户也越来越多地发起估计超过 30 分钟甚至数小时的人类工作量任务。这个案例凸显 agent 工具正在扩展员工跨职能完成技术和知识工作的边界。 OpenAI 文章称 agentic AI 正把知识工作的单位从单次互动转为可委托的长周期任务，Codex 在 OpenAI 内部已成为各部门的主要 AI 工具。到 2026 年 5 月，80.6% 的抽样个人用户至少提出过一个估计超过 30 分钟人工工作量的 Codex 请求，非开发者采用增长尤其快；OpenAI 认为这预示 agent 会扩展员工跨任务边界完成工作的能力。

6. Google Finance 推出投资组合与应用

  • 来源: Google AI Blog
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: Google Finance 投资组合 AI 研究 金融应用

摘要: 新版 Google Finance 结束测试并在全球推出投资组合功能，可整合持仓、展示表现和资产配置洞察，还支持用截图、CSV、PDF 或自然语言导入资产。它新增可定时运行的市场情报任务，并发布 Android 应用，提供自选股、实时数据、新闻流、AI 研究工具和股价异动解释。

实践建议

  • 试用投资组合导入能力，评估截图、CSV 和自然语言录入的准确性。
  • 设置一个每日盘前简报任务，观察其对自选股和持仓的个性化程度。
