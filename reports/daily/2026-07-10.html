1. GPT-5.6 发布：Sol、Terra、Luna 三档模型上线

  • 分类: 模型发布
  • 重要性: ⭐⭐⭐⭐⭐ (5/5)

摘要: OpenAI 发布 GPT-5.6 系列，将 Sol、Terra、Luna 分别定位为旗舰、日常均衡和高性价比模型，并强调更高的每美元任务完成效率。官方线程补充了 ChatGPT、Codex 与 API 的全球 rollout、办公产物质量提升，以及企业成本下降等重点。Microsoft 365 Copilot 也把 GPT-5.6 设为首选模型，说明这次发布不只是模型榜单更新，而是直接进入主流生产力套件。Sam Altman 的发布会转发和应用案例进一步把 GPT-5.6 与 ChatGPT Work、桌面端和 Sites 等产品线绑定在一起。

2. Meta 发布 Muse Spark 1.1 与 Model API 预览

  • 来源: Meta AI (Twitter)
  • 分类: 模型发布
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Muse Spark 1.1 Meta AI Model API 多模态 编码基准 代理式电脑使用

摘要: Meta AI 发布 Muse Spark 1.1，并同步推出 Meta Model API 公共预览，让开发者可以调用这款升级后的多模态模型。线程强调它已用于 Meta 内部编码和研究流程，在内部编码基准上具备竞争力，并强化感知、多模态推理、视觉到代码、视频理解和代理式电脑使用能力。后续早期反馈显示 Meta 正把 Muse Spark 作为模型能力、API 分发和 Meta AI 应用体验的共同升级。

📎 多角度报道:

  • Muse Spark 1.1 已用于 Meta 编码研究 · Meta AI (Twitter) · 编码研究
  • Meta 展示 Muse Spark 1.1 早期反馈 · Meta AI (Twitter) · 早期反馈
  • 扎克伯格称 Muse Spark 1.1 已上线 · Meta AI (Twitter) · Zuckerberg 转发

实践建议

  • 在视觉到代码、视频理解和浏览器操作任务上评估 Muse Spark 1.1。
  • 关注 Meta Model API 的速率限制、工具调用和多模态输入格式，判断是否适合集成。

3. ChatGPT Work 上线：Codex 驱动跨应用长任务

  • 来源: OpenAI Blog
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: ChatGPT Work Codex GPT-5.6 跨应用代理 桌面端 企业工作流

摘要: OpenAI 推出 ChatGPT Work，把 Codex 技术嵌入 ChatGPT，使其可以跨应用和文件持续数小时推进复杂项目，并产出文档、表格、演示、站点和报告等成品。官方线程说明了它由 GPT-5.6 驱动，支持 Web、移动端和新的 ChatGPT 桌面端，并按 Pro、Enterprise、Edu、Plus、Business 等套餐分阶段开放。Sam Altman 的转发强调 Codex 将从开发者工具扩展为更通用的工作代理。

📎 多角度报道:

  • OpenAI 发布 Codex 驱动的 ChatGPT Work · OpenAI (Twitter) · 官方发布
  • Sam 确认 Codex 将支撑 ChatGPT Work · Sam Altman (Twitter) · Codex 侧重

实践建议

  • 选择一个跨文档、表格和网页的真实项目测试 ChatGPT Work 的长任务稳定性。
  • 为代理访问文件和应用设置权限边界、审计日志与人工验收点。
  • 对比 Codex 独立应用与 ChatGPT Work 的任务拆解、上下文和交付物质量。

6. OpenAI Bio Bounty 转为持续私有计划

  • 来源: OpenAI Blog
  • 分类: 政策与安全
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: 生物安全 漏洞赏金 越狱测试 GPT-5.6

摘要: OpenAI 将 GPT-5.5 Bio Bug Bounty 演进为持续运行的私有 OpenAI Bio Bounty Program，重点寻找可绕过生物安全挑战的通用越狱。GPT-5.6 与 GPT-5.5 的通用越狱奖励提高到 5 万美元，GPT-5.5 原计划测试范围将持续到 2026 年 7 月 27 日。
