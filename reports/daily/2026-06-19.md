### 3. [Claude MCP加入企业托管认证](https://x.com/ClaudeDevs/status/2067655887662272723)

- **来源**: Claude Devs (Twitter)
- **分类**: 产品与功能
- **重要性**: ⭐⭐⭐⭐ (4/5)
- **标签**: `MCP` `企业认证` `集中授权` `Okta` `连接器治理` `Claude`

**摘要**: Claude Devs宣布MCP支持「Enterprise-Managed Auth」扩展，企业管理员可以通过身份提供商集中授权组织内的MCP连接器。该能力减少用户逐个应用OAuth和手动配置的摩擦，并让Claude聊天、Claude Code和Cowork之间的连接器访问保持一致。该扩展面向客户端、服务器和身份提供商开放，意味着企业AI工具的权限控制正在向协议层收敛。 Claude Devs称新方案通过企业身份提供商集中管理授权，用户不再需要为每个应用单独 OAuth 或设置。对管理员而言，这意味着更紧凑的访问控制和更高效的权限治理。 Claude Devs称企业托管认证已进入 beta，支持 Okta 以及 Asana、Atlassian、Canva、Figma、Granola、Linear、Supabase 等连接器，Slack 也将接入。该能力让 Claude 聊天、Claude Code 和 Cowork 之间的访问控制保持一致。 Claude Devs表示，企业托管认证建立在一个 MCP 扩展之上，任何客户端、服务器或身份提供商都可采用。该方向把企业身份和授权能力下沉到协议层，利于跨工具统一权限模型。

> **实践建议**
> - 评估现有MCP连接器是否适合改为集中授权，减少个人逐一配置。
> - 把连接器权限按团队和项目分层，并确认审计日志与撤权流程。
> - 构建MCP客户端或服务器时优先评估该认证扩展，避免自定义一次性授权方案。

### 8. [OpenAI发布生命科学评测LifeSciBench](https://x.com/OpenAI/status/2067346916929937827)

- **来源**: OpenAI (Twitter)
- **分类**: 研究
- **重要性**: ⭐⭐⭐⭐ (4/5)
- **标签**: `LifeSciBench` `生命科学` `GPT-Rosalind` `基准评测` `科研推理` `科研智能体`

**摘要**: OpenAI发布LifeSciBench，用750个由173位生物医药科学家编写的任务评估AI支持真实生命科学研究的能力，覆盖证据处理、分析、设计优化、转化和科学沟通等七类工作流。该基准强调模型能否基于证据推理、处理科研工件、表达不确定性并在现实约束下做出有用决策，而不只是回答生物知识题。结果显示GPT-Rosalind相较GPT-5.5在七个工作流上均有提升，但在复杂工件、设计密集任务和精确序列或结构输出上仍明显薄弱。 OpenAI补充称，LifeSciBench不只是测试生物知识或窄技能，而是衡量模型能否基于证据推理、处理科研工件、表达不确定性并在现实约束下做出有用决策。GPT-Rosalind在七个工作流上均超过GPT-5.5，但工件密集、设计密集和操作约束强的任务仍有较大改进空间。 OpenAI表示LifeSciBench将作为更真实评估、定向改进和生命科学社区协作的基础。其核心价值在于持续衡量进展、识别模型短板，并推动面向科研实际流程的AI能力改进。
