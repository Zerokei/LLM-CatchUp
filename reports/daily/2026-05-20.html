6. Claude Managed Agents 上线自托管沙箱与 MCP 隧道

  • 来源: Claude (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Claude Managed Agents 自托管沙箱 MCP 隧道 企业部署 MCP 上下文管理

摘要: 在「Code with Claude London」活动现场，Anthropic 为 Claude Managed Agents 发布两项安全能力：自托管沙箱（公开测试版）让 Agent 的执行环境运行在用户自有基础设施或托管沙箱提供商处，用户可在自有安全边界内运行 Agent 并默认套用自有安全策略；MCP 隧道（研究预览）则允许 Agent 访问私有网络内部的 MCP 服务器，而无需将其暴露到公网。此外 Managed Agents 还新增多项通用改进：可在不重启会话的情况下热替换工具、MCP 服务器或 vault ID，超过 10 万 token 的大型 MCP 工具输出会自动卸载到 Sandbox 文件以避免占满上下文。

📎 多角度报道:

  • Claude Managed Agents 通用改进：会话内热插工具与 MCP 大输出自动卸载 · Claude Devs (Twitter) · 工具热插拔等通用改进
  • MCP 隧道：Agent 无需暴露公网即可访问私有网络内的 MCP 服务器 · Claude (Twitter) · MCP 隧道能力详解
  • Claude Managed Agents 新增两项安全能力：自托管沙箱与 MCP 隧道 · Claude Devs (Twitter) · London 现场安全能力发布

实践建议

  • 对数据驻留/合规要求高的团队，用自托管沙箱在自有安全边界内运行 Agent
  • 通过 MCP 隧道让 Agent 安全访问内网服务，无需将服务暴露公网
  • 长会话中需切换工具或 MCP 配置时直接热替换，无需中断重启会话
  • 对会产生超大输出的 MCP 工具，依赖自动卸载到 Sandbox 文件控制上下文占用
