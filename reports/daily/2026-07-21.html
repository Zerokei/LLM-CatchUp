1. Claude Code 无障碍模式支持标注与提示音

  • 来源: Claude Devs (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Claude Code 无障碍 提示音 终端菜单

摘要: Claude Code 的屏幕阅读器模式增加 you:、claude:、tool: 等行标记，并把菜单改为可用数字选择，同时在 Claude 需要用户介入时发出提示音。用户可以在 ~/.claude/settings.json 中启用 axScreenReader，也可以设置对应环境变量持续开启。 Claude Code 团队发布屏幕阅读器模式的官方文档，补充说明了无障碍终端界面的使用方式。该条是同一功能发布线程的文档链接，帮助用户进一步配置和验证辅助技术支持。

实践建议

  • 在设置文件中启用 axScreenReader，检查行标记、数字菜单和等待提示音是否适合团队的辅助技术工作流。

2. Claude Code 额度问题仅短暂在线

  • 来源: Thariq (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Claude Code 短暂故障 额度异常 套餐

摘要: Thariq 补充说明，前述 Claude Code 问题只在线上存在了几分钟，他本人也在个人账号深夜编程时遇到过。该说明将事件定性为短暂故障，未表明 Claude Code 的套餐规则发生了正式变化。

3. OpenAI Sites 面向更多欧洲用户开放

  • 来源: OpenAI Devs (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: OpenAI Sites 欧洲可用性 快速原型 个人工具

摘要: OpenAI Devs 再次宣布 Sites 面向英国、欧洲经济区和瑞士的 Plus 与 Pro 账户开放，并展示了团队用它构建可发布、可分享的 Site。该线程强调了 Sites 从构想到个人应用原型的低门槛路径。 OpenAI Devs 在 Sites 可用性公告线程中补充了产品入口链接。它与同一线程的前文共同指向 Sites 的发布和分享能力，单条内容本身没有新增功能说明。

4. Claude Team 计划上手指南

  • 来源: Claude Devs (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: Claude Team 团队协作 共享项目 企业搜索

摘要: Anthropic 补充介绍 Claude Team 计划的使用方式，面向小团队提供共享项目、管理员控制、集中计费、SSO 和企业搜索。该计划起步席位从 5 个降至 2 个，降低了小团队采用协作功能的门槛。

实践建议
