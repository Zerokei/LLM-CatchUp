1. OpenAI Devs 官宣 GPT-5.5 API 发布

  • 来源: OpenAI Devs (Twitter) | 📡 也被 Sam Altman (Twitter), OpenAI (Twitter) 报道
  • 分类: 模型发布
  • 重要性: ⭐⭐⭐⭐⭐ (5/5)
  • 标签: GPT-5.5 OpenAI API 发布 token 效率 前沿模型

摘要: OpenAI Devs 正式宣布 GPT-5.5 在 API 中可用，称模型为复杂任务提供更高智能与更强 token 效率，能用更少重试完成任务。这是 GPT-5.5 在 ChatGPT/Codex 端推出后的 API 同步交付，开发者从此可直接接入。 OpenAI Devs 强调 GPT-5.5 在 agent 场景的提升：可自主规划、收集上下文、调用工具、从模糊状态中恢复，并在更少指导下完成更长流程，包括跨软件导航、跨应用执行动作以及多步代码与工具密集型任务。 GPT-5.5 已通过 Responses API 与 Chat Completions API 提供，支持 1M token 上下文窗口；GPT-5.5 Pro 仅在 Responses API 提供，面向高准确率任务。官方博客称 GPT-5.5 在 Terminal-Bench 2.0 达 82.7%、SWE-Bench Pro 58.6%，并在 Artificial Analysis Coding Index 上以前沿模型一半的成本提供 SOTA 表现；与 GPT-5.4 同等延迟下使用更少 token，已在 Cursor、NVIDIA、Axiom Bio 等早期合作方中验证。

实践建议

  • 立即在测试环境替换模型名为 gpt-5.5，跑全量 evals 对比 GPT-5.4 的成本/质量曲线
  • 把多步 agent 链中容易卡住的环节切到 GPT-5.5，验证重试率与超时率的下降幅度

2. DeepSeek-V4 Preview 开源发布：1.6T Pro / 284B Flash 双版本

  • 来源: DeepSeek (Twitter)
  • 分类: 模型发布
  • 重要性: ⭐⭐⭐⭐⭐ (5/5)
  • 标签: DeepSeek-V4 开源 MoE 1M 上下文 Flash

摘要: DeepSeek-V4 Preview 正式上线并开源，主打高性价比的 1M 上下文。DeepSeek-V4-Pro 为 1.6T 总参数 / 49B 激活，性能对标全球顶级闭源模型；DeepSeek-V4-Flash 为 284B 总 / 13B 激活，定位快速经济型。用户可通过 chat.deepseek.com 的 Expert / Instant 模式试用，API 当日更新可用，权重与技术报告同步发布在 Hugging Face。

实践建议

  • 在 Hugging Face 下载 DeepSeek-V4-Pro / Flash 权重，在自有评测集上验证与现有闭源模型的差距
  • 针对长文档 RAG 或代码库理解等场景，基于 1M 上下文重新设计提示与分块策略
  • 对比 Pro 与 Flash 在延迟与成本上的权衡，按任务复杂度路由到合适的版本

3. GPT-5.5 API 定价：输入 $5/M、输出 $30/M，1M 上下文窗口

  • 来源: Sam Altman (Twitter)
  • 分类: 模型发布
  • 重要性: ⭐⭐⭐⭐⭐ (5/5)
  • 标签: GPT-5.5 API定价 OpenAI 百万上下文 token效率

摘要: Sam Altman 公布 GPT-5.5 API 定价：每百万输入 token 5 美元、每百万输出 token 30 美元，并提供 100 万 token 的上下文窗口。他特别提示由于新模型在每个任务上消耗的 token 比 5.4 显著更少，实际单任务成本会低于按定价直接折算的结果。

实践建议

  • 对比 GPT-5.4 与 GPT-5.5 在同一任务上的 token 消耗，验证「token 用量减少」是否能抵消单价上涨
  • 对长文档/代码库分析任务，评估迁移到 GPT-5.5 的 1M 上下文窗口是否可去掉自建 RAG 切片

  • 来源: DeepSeek (Twitter)
  • 分类: 产品与功能

实践建议

  • 将现有 DeepSeek API 调用的 model 字段切换为 deepseek-v4-pro 或 deepseek-v4-flash 以提前测试兼容性
  • 在 7 月 24 日下线前完成 deepseek-chat / deepseek-reasoner 的迁移与回归测试
  • 评估 1M 上下文与 Thinking 双模式对长文档摘要与复杂推理场景的成本收益

  • 分类: 产品与功能
  • 重要性: ⭐⭐ (2/5)
  • 标签: Claude Code claude-api skill Managed Agents 记忆
