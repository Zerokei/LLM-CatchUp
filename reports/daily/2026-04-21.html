1. Anthropic × Amazon 扩大合作：锁定最多 5GW 算力，Amazon 追加投资 50 亿美元

  • 来源: Anthropic (Twitter)
  • 分类: 商业动态
  • 重要性: ⭐⭐⭐⭐⭐ (5/5)
  • 标签: Anthropic Amazon AWS 算力基建 融资

摘要: Anthropic 官方宣布与 Amazon 扩大合作：本季度起逐步上线最多 5GW 的训练与部署算力，2026 年底前约 1GW 到位；同时 Amazon 本日再追加 50 亿美元投资，未来还将最多追加 200 亿美元。公告延续了 Anthropic 在 Claude Opus 4.7 发布后持续扩张算力底座的主线，也把其与 Amazon 的绑定进一步深化到同等于 OpenAI 依赖微软的量级。

2. Claude Cowork 推出 Live Artifacts：可连接应用与文件的实时仪表盘与追踪面板

  • 来源: Claude (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Claude Cowork Live Artifacts 仪表盘 版本历史

摘要: Claude 官方宣布在 Cowork 中新增 Live Artifacts：用户可让 Claude 生成连接到自有应用与文件的仪表盘、追踪面板等交互式制品，重新打开时会以当前数据自动刷新。所有制品保存在新的 Live Artifacts 标签页并带版本历史，跨会话可持续迭代；功能已对全部付费计划开放。

实践建议

  • 更新 Claude 桌面或移动端到最新版本，进入 Cowork 侧边栏尝试让 Claude 生成一个连接 Notion / Google Sheet 的实时追踪面板。
  • 在 Live Artifacts 标签页建立一个长期项目的"主控台"原型，利用版本历史逐日迭代结构，验证其作为轻量 BI 的可用边界。

3. OpenAI Codex 推出 Chronicle：用屏幕上下文自动生成"记忆"（macOS Pro 试点）

  • 来源: OpenAI Devs (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: OpenAI Codex Chronicle 记忆系统 隐私

摘要: OpenAI Devs 扩展了上周预览的 Codex memories 功能，新增 Chronicle：由后台 Agent 读取近期屏幕截图自动沉淀工作上下文，Codex 不再需要用户逐次复述。Chronicle 会较快消耗速率限制；屏幕截图临时保存在本机、生成记忆后可自检查与编辑，但官方提醒其他应用也可能访问这些本地文件。先期仅对 macOS 上的 Pro 用户开放，欧盟、英国、瑞士暂不纳入。

实践建议

  • 若在 macOS 上使用 Codex Pro，升级后在设置中启用 Chronicle，观察它对跨会话上下文延续的实际帮助。
  • 开启后定期在记忆面板审查/删除条目，避免包含敏感仓库名、凭据或客户信息；同时评估速率消耗是否挤占常规使用。

4. Qwen3.6-Max-Preview：阿里下一代旗舰模型早期预览开放体验

  • 来源: Qwen (Twitter)
  • 分类: 模型发布
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Qwen Qwen3.6-Max 中国大模型 Agentic Coding 旗舰模型

摘要: 阿里通义千问放出下一代旗舰 Qwen3.6-Max-Preview 早期预览：相对 Qwen3.6-Plus 在 agentic coding 能力上有明显提升，世界知识与指令跟随更强，真实世界 Agent 与知识可靠性表现更稳。官方同步开放 Qwen Studio 与 API 入口，并预告 Qwen3.6 系列还将有更多型号跟进。

实践建议

  • 通过 Qwen Studio 或 API 跑一组你已有的 agentic coding 基准（如 SWE-bench 片段、工具调用链），横向对比 Qwen3.6-Plus 与 GPT-5.4 / Claude Opus 4.7 的差距。
  • 若已经在生产中集成 Qwen3.6-Plus，先在影子流量上并行 Max-Preview，关注指令跟随与工具可靠性的回归变化再决定是否切流。
