### 1. [Claude 不同模型与语言中的价值差异](https://www.anthropic.com/research/claude-values-models-languages)

- **来源**: Anthropic Research
- **分类**: 研究
- **重要性**: ⭐⭐⭐⭐ (4/5)
- **标签**: `Claude` `价值对齐` `多语言` `模型行为`

**摘要**: Anthropic 基于 309,815 条匿名 Claude.ai 对话，将主观问题中的价值表达压缩为四条可解释轴：服从与谨慎、温暖与严谨、深度与简洁、坦率与执行。研究发现 Sonnet 4.6 更偏服从、温暖和简洁，Opus 4.7 更偏谨慎、严谨、深入和坦率，不同语言间也存在明显差异。论文提出一种隐私保护监测方法，用于判断这些跨模型、跨语言差异是否符合预期或需要在训练中修正。

### 2. [Claude 延长 Fable 5 付费访问与 Code 额度](https://x.com/claudeai/status/2076351399999557669)

- **来源**: Claude (Twitter)
- **分类**: 产品与功能
- **重要性**: ⭐⭐⭐⭐ (4/5)
- **标签**: `Claude Fable 5` `Claude Code` `付费计划` `使用额度`

**摘要**: Claude 宣布将所有付费计划中的 Claude Fable 5 访问期延长至 7 月 19 日，并继续把 Claude Code 每周速率限制维持在提高 50% 的水平。这属于面向付费用户和 Claude Code 工作流的临时额度与访问政策更新。 Claude 补充说明，用户最多可将每周使用额度的一半用于 Fable 5；超过后可继续用使用积分访问 Fable 5，或切换到其他模型以消耗剩余额度。该说明明确了促销访问期内 Fable 5 的额度边界和续用方式。

> **实践建议**
> - 检查团队付费账户是否可在 7 月 19 日前优先跑完 Fable 5 评测任务。
> - 把 Claude Code 高额度窗口用于迁移、重构或长链路调试等高消耗任务。
