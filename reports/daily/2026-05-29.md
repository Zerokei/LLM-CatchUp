### 8. [OpenAI API 平台引入 Workload Identity Federation](https://x.com/OpenAIDevs/status/2059703600662925635)

- **来源**: OpenAI Devs (Twitter)
- **分类**: 产品与功能
- **重要性**: ⭐⭐⭐ (3/5)
- **标签**: `OpenAI` `WIF` `IAM` `API 安全` `凭证管理`

**摘要**: OpenAI 在 API 平台引入 Workload Identity Federation（WIF），允许团队通过云端 IAM 工作流管理对 OpenAI 的访问权限，减少在各服务间分发长期有效 API Key 的需要。此举显著降低 Key 泄漏与轮换的运维负担，对接 AWS、GCP、Azure 等云身份体系。 OpenAI 扩展 Admin API，新增针对企业项目的可编程管理能力，包括支出告警、模型 allowlist、数据保留控制、托管工具控制，以及对 file search、web search 等能力更细粒度的成本可见性。该更新便于大型团队按项目维度统一治理与审计 OpenAI 使用情况。

> **实践建议**
> - 用 Workload Identity Federation 替换硬编码的 OpenAI API Key，统一接入云端 IAM
> - 为生产环境的 OpenAI 调用启用短期联邦凭证，结合现有审计日志监控异常访问
