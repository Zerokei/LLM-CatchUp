### 2. [Claude Code 推出 Routines：可调度的长驻自动化任务](https://x.com/claudeai/status/2044095086460309790)

- **来源**: Claude (Twitter)
- **分类**: 产品与功能
- **重要性**: ⭐⭐⭐⭐ (4/5)
- **标签**: `Claude Code` `Routines` `自动化` `调度` `Web 基础设施`

**摘要**: Claude Code 以研究预览形式推出 Routines——一次配置 prompt、代码仓库与连接器即可按定时、API 调用或事件驱动反复运行。任务运行在 Anthropic 的 Web 基础设施上，无需保持本机开启；已适用于全部付费计划，CLI 中的 `/schedule` 自动迁移为 Routines。

> **实践建议**
> - 创建一个 Routine：每天凌晨 2 点从 Linear 拉取最高优先级 bug，尝试修复并开 draft PR，观察一周后的命中率
> - 将团队已有的 `/schedule` CLI 作业检查一遍，确认迁移后仍按预期触发，同时接入事件驱动模式做对比

### 6. [Chrome 推出 Skills：把常用 AI prompt 一键化并可复用](https://blog.google/products-and-platforms/products/chrome/skills-in-chrome/)

- **标签**: `Chrome` `Gemini` `Skills` `提示工程` `浏览器 AI`

**摘要**: Google 在 Chrome 中推出 Skills 功能，用户可以发现、保存并 remix 他人的 AI 工作流，把高质量 prompt 固化为一键触发的工具并在浏览器中随时复用。该功能瞄准重复性的 AI 使用场景，降低 prompt 工程门槛。

> **实践建议**
> - 在 Chrome 中开启 Skills，保存 2-3 个你最常用的日常 AI prompt（翻译、摘要、代码解释），比较一键调用与手动粘贴的效率差
