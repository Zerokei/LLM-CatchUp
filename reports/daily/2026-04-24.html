1. OpenAI 发布 GPT-5.5：最强 Agentic 编码与知识工作模型正式推出

  • 来源: OpenAI Devs (Twitter)
  • 分类: 模型发布
  • 重要性: ⭐⭐⭐⭐⭐ (5/5)
  • 标签: GPT-5.5 模型发布 Agentic编码 计算机操控 科学研究

摘要: OpenAI 正式发布 GPT-5.5，称其为迄今最智能、最直觉化的模型，专为 Agentic 编码、计算机操控、知识工作和科学研究而优化。GPT-5.5 在 Terminal-Bench 2.0 达到 82.7%（SOTA），OSWorld-Verified 达到 78.7%，GDPval 达到 84.9%，同时以与 GPT-5.4 相当的延迟提供显著更高的智能水平，并以更少 token 完成 Codex 任务。目前已向 Plus、Pro、Business 和 Enterprise 用户在 ChatGPT 和 Codex 中推出，API 版本即将上线。GPT-5.5 还在数学（拉姆齐数新证明）和生物信息学基准（BixBench）上展示了科研辅助能力，并采用了迄今最强的网络安全防护措施。

实践建议

  • 立即在 ChatGPT Plus/Pro 账号中切换至 GPT-5.5 试用 Agentic 编码任务，重点测试长上下文代码调试和多步骤重构场景
  • 通过 Codex 使用 GPT-5.5 处理跨工具、跨文件的复杂工程任务，利用其更强的系统理解能力减少人工干预
  • 等待 API 版本上线后，将 GPT-5.5 集成至生产工作流，评估其在实际业务场景（如文档处理、数据分析）的 ROI

2. Claude Managed Agents 记忆可导出为文件并通过 API 管理

  • 来源: Claude Devs (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Managed Agents Memory API 可移植性 持久化

摘要: Anthropic 披露 Claude Managed Agents 的 Memory 功能支持将记忆内容导出为可移植文件，开发者可通过 API 直接管理记忆存储，具备完整的编程控制能力。记忆存储以工作区为单位，挂载为容器内的文本文件，跨会话持久保留。相关文档已发布于 platform.claude.com/docs/en/managed-agents/memory。 Anthropic 为 Claude Code 内置的 claude-api skill 添加了对 Managed Agents Memory 的支持，开发者可通过触发 /claude-api 命令并询问 Memory 相关问题来获取即时指引。该 skill 的源码已开放在 GitHub anthropics/skills 仓库，方便二次定制。这一更新降低了开发者在 Claude Code 环境中探索 Memory API 的门槛。 Claude Managed Agents 的 Memory 以工作区为作用域，底层存储为挂载在容器内的文本文件，Session 结束后数据仍持久保留。当 Memory Store 挂载到会话时，Claude 可在会话中直接读写这些文件，实现跨对话的信息积累。该机制为长期运行的 Agent 提供了原生持久化能力。

实践建议

  • 查阅 platform.claude.com/docs/en/managed-agents/memory 文档，了解如何通过 API 创建、读取和管理记忆存储。
  • 在 Agent 应用中集成 Memory API，实现跨会话的上下文持久化，减少重复输入用户偏好和历史信息的开销。
  • 利用记忆导出功能定期备份 Agent 学到的关键信息，或在不同工作区之间迁移记忆数据。

3. GPT-5.5速度与GPT-5.4持平，每任务所需token大幅减少

  • 来源: Sam Altman (Twitter)
  • 分类: 模型发布
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: GPT-5.5 推理效率 token优化 ChatGPT Codex

摘要: Altman介绍GPT-5.5的性能特性：每token速度与GPT-5.4相当，但完成相同任务所需token数显著更少，整体效率大幅提升。模型今日起在ChatGPT和Codex上线，API版本正与客户协调安全与防护事宜后即将推出。Altman用「懂得该做什么」描述其直觉般的任务理解能力。 Altman公布GPT-5.5的API定价：输入$5/百万token，输出$30/百万token，上下文窗口达100万token。他特别提醒，由于GPT-5.5完成任务所需token更少，实际成本会低于GPT-5.4。高达百万token的超长上下文是重要亮点，为处理大规模文档和复杂多轮任务提供了基础。

实践建议

  • 在相同任务上对比GPT-5.4与GPT-5.5的token消耗，量化效率提升以评估迁移收益
  • 关注Codex中GPT-5.5的代码生成能力，测试复杂编程任务的完成质量
