3. Claude Artifacts 接入 MCP 连接器

  • 来源: Claude Devs (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Claude Artifacts MCP 连接器 实时数据 权限控制

摘要: Claude Code artifacts 现在可以调用 MCP connectors，让仪表盘和应用按查看者自己的权限即时拉取数据并执行动作。创作者只需构建一次工件，后续不必重新运行会话刷新内容，查看者也只能访问自己有权访问的数据。该能力面向 Pro、Max、Team 和 Enterprise 计划开放，但不适用于公开分享的 artifacts。 Claude Devs 介绍了一项 Artifacts 能力：创作者构建一次工件后，工件可通过观看者自己的 MCP 连接器拉取实时数据。这样创作者无需重新运行会话刷新内容，同时观看者只能访问自己有权限的数据。

实践建议

  • 把现有静态 artifact 改造成连接只读 MCP 数据源的仪表盘，先验证权限边界。
  • 避免把需要公开分享的 artifact 依赖 MCP 连接器，因为该能力不支持公开分享。

5. GPT-Red：自动红队推动安全自改进

  • 来源: OpenAI Blog
  • 分类: 研究
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: GPT-Red 自动红队 提示注入 GPT-5.6 模型安全 自博弈

摘要: OpenAI 发布内部自动红队系统 GPT-Red，用自博弈强化学习生成提示注入等攻击，在网页、邮件、工具输出和本地文件等现实场景中扩展红队覆盖。GPT-Red 在间接提示注入评测中显著超过人类红队，并能攻击真实代理系统与 Codex CLI 场景。OpenAI 已把 GPT-Red 生成的攻击用于 GPT-5.6 对抗训练，使 GPT-5.6 Sol 在最难直接提示注入基准上的失败率比四个月前最佳生产模型低 6 倍。

8. Codex in Chrome 演示上线计划流程

  • 来源: OpenAI Devs (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: Codex Chrome 工作流 连接器

摘要: OpenAI Devs 展示 Codex in Chrome 如何把一次请求转成上线计划：从表单生成清单，拉取 Google Drive、Slack 和本地文件上下文，标记待跟进事项，并更新门户和草拟回复。该演示强调 Codex 负责跨工具整理和执行，人类保留最终决策权。

实践建议

  • 选择一个真实上线请求，按表单到清单到回复的链路复刻演示流程。
  • 检查 Google Drive、Slack 和本地文件权限，确保 Codex 只读取必要上下文。
