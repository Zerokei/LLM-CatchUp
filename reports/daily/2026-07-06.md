### 1. [Claude Code 循环：从提示到自动化例行程序](https://baoyu.io/blog/2026-07-06/claudedevs-2074208949205881033)

- **来源**: 宝玉的分享
- **分类**: 产品与功能
- **重要性**: ⭐⭐⭐⭐ (4/5)
- **标签**: `Claude Code` `智能体循环` `自动化` `技能` `成本控制`

**摘要**: 文章把 Claude Code 中的循环定义为智能体重复执行工作周期，直到满足停止条件，并按触发方式、停止条件、基础指令和适用任务进行分类。它介绍了回合制循环、目标导向循环、基于时间的循环和主动式循环，强调用明确验证标准、技能文件和合适的触发频率控制成本与质量。

> **实践建议**
> - 把常见开发任务拆成回合制、目标导向或定时循环三类，分别选择普通提示、/goal 或 /loop。
> - 为高频任务编写 SKILL.md，把验收标准、测试命令和浏览器检查步骤固化为可重复验证。
> - 对长跑自动化先设定最大尝试次数和 token 预算，再扩大到更多任务。

### 3. [阿尔伯塔用 Claude 扫描 4.66 亿行政府代码](https://www.anthropic.com/news/alberta-government-claude-cybersecurity)

- **来源**: Anthropic Blog
- **分类**: 政策与安全
- **重要性**: ⭐⭐⭐ (3/5)
- **标签**: `Claude Code` `政府数字化` `网络安全` `遗留系统` `智能体`

**摘要**: 加拿大阿尔伯塔省政府自 2025 年起使用 Claude Code、Opus 和 Sonnet 审查政府系统，在约 20 小时内扫描 4.66 亿行代码，覆盖 1,280 个应用和 3,400 个代码库。团队用约 50 个并行智能体发现安全漏洞、基础设施弱点和文档缺口，并让 Claude 生成补丁、补测试或重构遗留系统，同时由工程师审查后再上线。
