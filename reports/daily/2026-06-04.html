1. Gemma 4 12B 发布：本地多模态模型升级

  • 来源: Demis Hassabis (Twitter)
  • 分类: 模型发布
  • 重要性: ⭐⭐⭐⭐⭐ (5/5)
  • 标签: Gemma 4 12B模型 多模态 本地部署 Apache 2.0

摘要: Demis Hassabis 转发庆祝 Gemma 下载量突破 1.5 亿，并介绍新的 Gemma 4 12B 模型。该模型采用统一、无编码器的多模态设计，面向本地笔记本运行场景，官方强调其在 16GB VRAM 设备上也足够轻量，并以 Apache 2.0 许可证发布。

实践建议

  • 需要端侧多模态能力的团队可把 Gemma 4 12B 加入 16GB VRAM 设备的基准测试。
  • 开源使用前确认 Apache 2.0 许可与内部模型分发规范是否匹配。
  • 用真实图文推理和本地延迟任务评估它相对云端模型的成本收益。

2. GPT-Rosalind 升级：GPT-5.5 进入生命科学

  • 来源: OpenAI Blog
  • 分类: 模型发布
  • 重要性: ⭐⭐⭐⭐⭐ (5/5)
  • 标签: GPT-Rosalind GPT-5.5 生命科学 药物发现 企业AI

摘要: OpenAI 为面向企业生命科学研究的 GPT-Rosalind 推出新能力，把 GPT-5.5 的智能体编码和工具调用能力与药物化学、基因组学、实验工作流等专业推理结合起来。更新后的模型在生物专家任务、复杂药物化学查询、定量生物学和湿实验排错中显示更强表现，并通过 trusted-access 结构向合格组织开放研究预览。官方社媒进一步强调这是把 GPT-5.5 用于药物发现、分析、设计和实验流程的垂直化落地。

📎 多角度报道:

  • OpenAI 社媒介绍 GPT-Rosalind 新能力 · OpenAI (Twitter) · 社媒要点

实践建议

  • 生命科学团队可优先整理药物化学、基因组学和实验排错任务集做封闭评测。
  • 评估时把工具调用、代码生成和领域专家审阅放进同一工作流，而不是只看单轮问答。
  • 对受监管研发数据使用 trusted-access 部署，先明确数据边界和审计要求。

3. Claude Code 更新：动态工作流、/fork 与自检

  • 来源: Claude Devs (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Claude Code 动态工作流 ultracode /fork 自检

摘要: Claude Code 相关更新集中在把编码代理从单次会话推进到可编排流程：动态工作流的触发词从 workflow 改为 ultracode，以减少普通语义中的误触发。/fork 现在可以用当前完整上下文、工具、历史、模型和提示缓存启动后台代理，并把结果返回原会话。配套内容还展示如何把人工检查编码成 Claude Code 的自检环节，让代理在交付前闭合反馈回路。

📎 多角度报道:

  • Thariq 解读 Claude Code 工作流实践 · Thariq (Twitter) · 实践解读
  • Claude Code 动态工作流长文入口 · Thariq (Twitter) · 长文入口
  • Claude Code 可编码人工自检流程 · Claude Devs (Twitter) · 自检流程

实践建议

  • 将常用复杂任务迁移到 ultracode 触发的动态工作流，并记录误触发样例。
  • 用 /fork 跑高风险改动的后台候选方案，主会话只审查结果和关键 diff。
  • 把人工验收清单写进 Claude Code 流程，让代理提交前先自查测试、格式和边界条件。

4. Anthropic 用 MITRE 映射 AI 网络威胁

  • 来源: Anthropic Blog
  • 分类: 政策与安全
  • 重要性: ⭐⭐⭐⭐ (4/5)

  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)

摘要: OpenAI 将 Codex 插件从单个工具扩展为面向具体角色的一键安装能力，让 Codex 可在销售、数据分析、创意制作、产品设计和公开股票投资等场景中变成专门助手。官方称这些插件可访问 62 个常用应用和 110 个技能，OpenAI Devs 进一步说明插件围绕团队真实工作流构建，并把 Sites 用作原型输出后的团队评审入口。示例包括销售准备、公开股权投资、数据分析、创意制作和产品设计。

  • OpenAI Devs 解释 Codex 角色插件 · OpenAI Devs (Twitter) · 开发者视角

实践建议

  • 按岗位挑选一个高频流程试装 Codex 角色插件，记录它实际调用的 app 和 skill。
