1. GPT-5.6 扩展落地：Copilot、医疗与编码

  • 来源: OpenAI (Twitter)
  • 分类: 模型发布
  • 重要性: ⭐⭐⭐⭐⭐ (5/5)
  • 标签: GPT-5.6 Microsoft 365 Copilot 医疗智能 Codex ChatGPT Work Luna

摘要: GPT-5.6 发布后的第二波信息集中在落地面：它已进入 Microsoft 365 Copilot 的优先模型路径，并被 OpenAI 用于强调健康智能场景。OpenAI 称 GPT-5.6 Luna 在最高推理设置下超过 GPT-5.5，同时成本低 25 倍，凸显旗舰能力向低成本使用扩散。开发者侧还出现近千行编码任务案例、Codex AMA 和临时额度重置，说明 OpenAI 正在把模型发布转化为 Work、Codex 与企业办公场景的实际试用。

  • Sam 称用户喜欢 GPT-5.6 Sol · Sam Altman (Twitter) · 用户反馈
  • GPT-5.6 Sol 发布后临时重置额度 · Sam Altman (Twitter) · 额度重置
  • GPT-5.6展示近千行编码任务能力 · OpenAI Devs (Twitter) · 编码案例
  • OpenAI开启GPT-5.6与Codex AMA · OpenAI Devs (Twitter) · Codex AMA

实践建议

  • 在 365 Copilot、Codex 和 Work 中分别挑选一个真实任务，对比 GPT-5.6 与旧模型的质量、耗时和成本。
  • 用医疗摘要、长文档处理和跨文件编码三类任务测试 Luna 与 Sol 的性价比分界。
  • 把临时额度窗口用于长链路试验，并记录需要人工接管的步骤。

2. Claude Code 桌面端加入内置浏览器

  • 来源: Claude Devs (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Claude Code 内置浏览器 网页交互 桌面应用 开发智能体

摘要: Claude Code 桌面版新增内置浏览器，Claude 可以打开文档、设计稿或其他网站，并像处理本地开发服务器一样阅读、点击和交互。该能力带有沙箱和可配置会话持久化选项，用户需要更新到最新版桌面应用并查阅外部站点浏览文档。对开发代理来说，这把查资料、看设计和验证网页状态纳入了同一个桌面工作流。 Claude Devs 提醒用户更新到最新版桌面应用，并指向关于浏览外部网站的文档。该条是同一线程的补充说明，重点是新桌面能力需要通过最新客户端获得，并可查阅配置与使用细节。

实践建议

  • 升级 Claude Code 桌面版后，用项目文档、设计稿和本地开发服务器各跑一次端到端验证。
  • 为团队定义允许持久会话的外部站点清单，避免代理浏览器保留不必要的登录状态。
  • 把需要查文档和点击验证的开发任务改写成可复用的 Claude Code 流程。

3. ChatGPT、Work、Codex 的任务边界

  • 来源: 宝玉的分享
  • 分类: 教程与观点
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: ChatGPT Codex Work 智能体额度

摘要: 文章用「Chat 回答问题，Work 帮你干活，Codex 帮你写代码」区分三种模式：Chat 面向问答与内容生成，Work 面向跨应用办公交付，Codex 面向仓库内开发任务。它还说明 Work 与 Codex 共用智能体额度、Chat 额度独立，并对比了 ChatGPT Work 与 Claude Cowork 在跨设备、内置浏览器和插件目录上的差异。

  • 来源: OpenAI (Twitter)
  • 分类: 政策与安全
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: OpenAI 生物安全 漏洞赏金 红队测试
