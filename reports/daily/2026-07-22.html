1. Gemini 3.6 Flash 与 3.5 系列模型发布

  • 来源: Google DeepMind (Twitter)
  • 分类: 模型发布
  • 重要性: ⭐⭐⭐⭐⭐ (5/5)
  • 标签: Gemini 3.6 Flash Gemini 3.5 Flash-Lite 模型发布 多模态 API

摘要: Google DeepMind 发布 Gemini 3.6 Flash 以及 Gemini 3.5 Flash、Flash-Lite 等模型，分别面向高质量推理、编程多模态任务和高吞吐低成本场景。新模型同步扩展到应用、API 和安全试点，开发者可以按延迟、质量与 token 效率选择部署层级。多条产品更新显示 Gemini 3.x 正在从单一模型升级为覆盖不同工作负载的系列化产品线。 Gemini 3.6 Flash 和 3.5 Flash-Lite 正在 Gemini App 中推出，开发者可通过 Google AI Studio 和 Android Studio 的 API 开始构建。面向网络安全的 Gemini 3.5 Flash Cyber 将先通过 CodeMender 以限量试点方式提供。

📎 多角度报道:

  • Gemini 3.5 Flash-Lite 面向高吞吐任务优化 · Google DeepMind (Twitter) · Flash-Lite 性能定位
  • Gemini 3.6 Flash 强调质量与 token 效率 · Google DeepMind (Twitter) · Flash 质量与 token 效率

实践建议

  • 用同一批编程、多模态和高吞吐任务分别评估 Flash 与 Flash-Lite 的质量、延迟和成本。
  • 检查现有 API 或应用接入路径，优先在低风险流量中验证新模型的兼容性和安全配置。

3. OpenAI Presence：面向企业的可信语音与聊天代理平台

  • 来源: OpenAI Blog
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: OpenAI Presence 企业代理 语音代理 护栏 Codex

摘要: OpenAI 推出 Presence，帮助企业部署能够回答问题、调用内部系统、执行获批操作并在必要时转人工的语音和聊天代理。平台将权限策略、护栏、模拟测试、评估工具和 Codex 驱动的持续改进流程整合在一起，目前通过限量通用可用计划向符合条件的企业提供。

实践建议

  • 先选定一个边界清晰的高价值流程，限定代理可访问的知识库、系统和获批动作。
  • 用模拟场景覆盖常见请求、边界案例和高风险操作，再设置人工升级规则后逐步放量。
  • 持续收集生产会话与转人工信号，用评估结果验证每次策略或提示变更。

  • 来源: Claude Devs (Twitter)
  • 分类: 产品与功能

11. Claude 接入 Anthropic Economic Index 数据连接器

  • 来源: Anthropic Blog
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: Claude Economic Index 连接器 AI 与就业 数据分析

摘要: Anthropic 为 Claude 推出 Economic Index 连接器，用户可以直接询问不同行业、职业和地区如何使用 AI，并获得基于 Index 数据的回答。连接器无需安装，在 claude.ai 的连接器目录中启用即可，但数据反映的是 Claude 使用模式而非完整劳动力市场，回答会保留这一限制说明。

实践建议

  • 先用连接器查询本行业的 AI 使用任务，再追问数据来源、样本范围和时间变化。
  • 把连接器结果与本组织的实际任务数据分开记录，避免将 Claude 使用模式直接当作整体劳动力市场结论。

13. Karpathy：用语音长聊帮助 LLM 理清复杂意图

  • 来源: Andrej Karpathy (Twitter)
  • 分类: 教程与观点
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: Andrej Karpathy 语音输入 提示工程 长上下文

摘要: Andrej Karpathy 分享了一种与大语言模型协作的方式：开启语音输入，连续讲述约十分钟的杂乱想法，再让模型重构并整理内容。较长的上下文能减少反复澄清，让模型更准确地捕捉用户目标，也帮助用户看清自己的思路。

14. Codex 侧边对话与子代理状态更易追踪

  • 来源: OpenAI Devs (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: Codex 子代理 侧边对话 任务状态

摘要: Codex 现在更快显示侧边对话和子代理的活动状态，帮助用户及时了解并行任务的进展。排队中的后续任务也可以直接转为侧边对话，无需复制粘贴上下文。 Codex 更新了任务切换时保留草稿、快速返回搜索位置以及可点击路径面包屑等细节。预览会在对应任务中打开，自动化则遵循本地时间运行，减少跨任务操作中的上下文丢失。

实践建议

  • 将可并行的调研或代码任务拆给子代理，并在侧边对话中分别跟踪进度。
  • 把排队中的后续任务转成侧边对话，保留任务上下文并减少手工复制。
