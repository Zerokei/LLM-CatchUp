1. Kimi K3 发布：2.8T 开放模型 + 百万上下文

  • 来源: Kimi (Twitter)
  • 分类: 模型发布
  • 重要性: ⭐⭐⭐⭐⭐ (5/5)
  • 标签: Kimi K3 开放模型 2.8T 参数 百万上下文 多模态 Agent 编程 前端评测

摘要: Kimi K3 是 Moonshot 发布的 2.8T 参数开放 3T 级模型，具备原生视觉、100 万 token 上下文和长程代理编码能力。官方技术博客披露其基于 Kimi Delta Attention、Attention Residuals 和高稀疏 MoE 架构，并展示 GPU kernel 优化、MiniTriton 编译器、芯片设计、研究可视化与视频编辑案例。Kimi K3 已在 Kimi.com、Kimi Work、Kimi Code 和 API 开放，完整权重计划于 2026 年 7 月 27 日发布。第三方榜单称它在 Frontend Code Arena 登顶，强化了其前端生成和视觉闭环编程的发布叙事。

📎 多角度报道:

  • Kimi K3 强调视觉闭环编程 · Kimi (Twitter) · 技术博客与视觉闭环
  • Kimi K3 登顶 Frontend Code Arena · Kimi (Twitter) · 前端榜单登顶
  • Kimi 发布 K3 预告入口 · Kimi (Twitter) · 发布预告入口

实践建议

  • 先通过 Kimi API 或 Kimi Code 验证长程编码、前端生成和研究自动化任务。
  • 权重发布后复现实测项目，对比它与现有 Claude/GPT 工作流的差异。
  • 评估官方推理建议和 vLLM 支持，再决定是否进行本地或私有化部署。

2. Claude Code 代码审查升级：effort 分级 + ultra 复核

  • 来源: Claude Devs (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Claude Code 代码审查 effort 分级 多代理 PR 协作 误报控制

摘要: Claude Code 的 /code-review 现在按 effort 档位重写审查策略，让团队在 token 成本、召回率和深度之间选择。低档主打低成本发现问题，高档用于更深检查，ultra 则会启动多个审查代理并独立复现发现以降低误报。该功能已在所有 Claude Code 会话中开放，更新后即可在 PR 中运行。

📎 多角度报道:

  • Claude Code 推出 ultra 审查档位 · Claude Devs (Twitter) · ultra 多代理复核

实践建议

  • 为普通 PR 默认使用低档，为核心路径或发布前改动使用 high/ultra。
  • 用历史 PR 对比不同档位，建立团队自己的成本、召回率和误报基线。
  • 把 /code-review 输出纳入合并前清单，并保留人工复核环节。

3. Google Search AI Mode 接入外部应用

  • 来源: Google AI Blog
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Google Search AI Mode 应用连接 Personal Intelligence 任务执行

摘要: Google 开始在美国推出 Search AI Mode 的 connected apps，让用户可安全连接并直接操作常用服务。首批场景包括把烧烤购物清单加入 Instacart 购物车、用 Canva 查找设计模板，以及把派对歌单保存到 YouTube Music。该能力把搜索从回答问题推进到跨应用执行任务，也会让 Personal Intelligence 的回答更贴合用户上下文。

实践建议

  • 测试 AI Mode 连接应用时的授权、数据边界和撤销流程
  • 把 Instacart、Canva、YouTube Music 场景拆成可复用的产品集成清单
  • 关注后续伙伴扩展，评估自家服务是否需要适配搜索入口

4. Codex 增加 PR Chat 与行内改稿

  • 来源: OpenAI Devs (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: Codex 代码评审 PR 协作 开发工具

摘要: OpenAI Devs 介绍 Codex 的 PR Chat，可在具体拉取请求上下文中提问并追踪后续修改。新的行内代码编辑让评审反馈可以直接交给 Codex，用户能在同一界面检查补丁并选择编辑、接受或拒绝。

  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: Muse Spark OpenRouter Meta AI 开发者接入

7. Google Vids 加入 Gemini Omni 与头像

  • 来源: Google AI Blog
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: Google Vids Gemini Omni 数字头像 视频生成 Workspace

摘要: Google Vids 推出两项更新：Gemini Omni 可通过文本提示和图像参考生成、编辑高质量视频片段，personal avatars 则让用户上传自拍和语音录制来创建数字分身出镜。用户可以用自然语言换背景、修正光线或生成片段，减少传统视频制作中的拍摄和剪辑成本。所有 AI 生成视频都会加入数字水印，以提高内容透明度。

实践建议

  • 用一段内部培训视频测试文本改片、图像参考和头像出镜的实际质量
  • 为 AI 生成视频建立水印、审核和发布前告知规范
