1. Anthropic Labs 发布 Claude Design：用对话生成原型、幻灯片与一页纸

  • 来源: Claude (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Claude Design Opus 4.7 视觉模型 原型设计 Anthropic Labs

摘要: Anthropic Labs 推出 Claude Design，由迄今最强视觉模型 Claude Opus 4.7 驱动，可通过对话生成原型、幻灯片和一页纸，支持内联评论、直接编辑和自定义滑块进行精修，导出 PDF/PPTX 或交付给 @canva、Claude Code。Claude 还能读取代码库与设计文件构建团队设计系统并自动应用，确保品牌一致性。该功能在 Pro、Max、Team 和 Enterprise 计划的研究预览中陆续开放。

实践建议

  • 在 Pro/Max/Team/Enterprise 计划中打开 Claude，找到 Claude Design 入口，尝试用一句话生成一份产品 Pitch 一页纸
  • 上传现有品牌色板/Logo/字体规范，让 Claude 提取并复用为团队设计系统，再生成一组与之一致的幻灯片
  • 完成原型后导出为 PPTX 或交给 Claude Code，把设计转译为可运行的前端代码

2. xAI 推出 Grok Speech-to-Text API：25 种语言的多说话人转写

  • 来源: xAI (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐⭐ (4/5)
  • 标签: Grok 语音识别 STT API 多语言 多说话人

摘要: xAI 上线 Grok Speech-to-Text API，支持 25 种语言的即时多说话人转写，并以"市场最低价"作为定价主张。这是 Grok 在文本与图像之外补齐音频输入能力的关键一步，使其更接近完整的多模态 API 套件。

实践建议

  • 申请 xAI API Key 后调用 STT 端点测试一段中文/英文混合的会议录音，验证多说话人分离效果
  • 将 Grok STT 与现有 OpenAI Whisper / Gemini 语音方案做横向对比，重点关注中文和长音频的准确率与价格
  • 将转写结果直接喂给 Grok 文本模型形成"语音 → 摘要"流水线，评估端到端延迟

3. Claude for Word 上线 Pro 与 Max 计划，与 Opus 4.7 协同工作

  • 来源: Claude (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: Claude for Word Office 集成 Opus 4.7 办公 生产力

摘要: Anthropic 将 Claude for Word 推送至 Pro 与 Max 订阅用户，并与最新的 Opus 4.7 模型直接联动。这意味着用户可以在 Word 文档中直接调用 Claude 完成起草、改写和长文档分析，与上月发布的 Claude for Excel 共同补齐 Anthropic 在 Microsoft 365 生产力栈中的覆盖。

实践建议

  • 在 Word 加载项市场安装 Claude for Word，绑定 Pro/Max 账号，用 Opus 4.7 重写一段长报告测试编辑质量
  • 让 Claude for Word 对照公司模板检查用词与结构一致性，作为"AI 校稿员"嵌入日常文档流程
  • 与 Claude for Excel 配合使用：先在 Excel 用 Claude 做数据汇总，再在 Word 用 Claude 生成解读段落

4. Claude Code Hackathon 重启：面向 Opus 4.7、奖池 10 万美元 API 额度

  • 来源: Claude (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: Claude Code 黑客松 Opus 4.7 开发者社区 Anthropic

摘要: Anthropic 宣布 Claude Code Hackathon 为 Opus 4.7 重启，邀请全球开发者与 Claude Code 团队同处一周，奖池为 10 万美元 API 额度，申请截止本周日。这是继 @ClaudeDevs 账号开通后 Anthropic 持续加码 Claude Code 开发者生态的明确信号。

实践建议

  • 在截止前提交申请，用一段简短的 "用 Claude Code 做了什么 + 想做什么" 提案
  • 即使不参赛，也可关注后续公开的获奖项目，作为 Claude Code agentic 工作流的参考案例
  • 将 Opus 4.7 的新能力（如改进的工具调用、代理规划）纳入项目设计，提高入选概率

5. Gemini 3.1 Flash 文本转语音登陆 AI Studio，可触发指定语气

  • 来源: Google DeepMind (Twitter)
  • 分类: 产品与功能
  • 重要性: ⭐⭐⭐ (3/5)
  • 标签: Gemini 3.1 Flash TTS AI Studio 语音合成 Google

摘要: Google DeepMind 转推 Google AI Studio：Gemini 3.1 Flash 的文本转语音能力上线 AI Studio，可触发特定的语音传递方式（如语调、情感）。Flash 系列定位是低延迟、低价位档，加上可控的语音风格，使其在客服 IVR、智能播报等实时音频场景具备直接竞争力，与同日 xAI Grok STT 的发布形成"音频输入 + 输出"两端齐发的格局。

实践建议

  • 打开 Google AI Studio，选择 Gemini 3.1 Flash，尝试在 prompt 中显式指定语气（如"用兴奋、急促的口吻播报"）观察音色变化
  • 将 Gemini 3.1 Flash TTS 与 ElevenLabs / OpenAI TTS 做对照，重点比较中文语调自然度和延迟
  • 把可控语气 TTS 接入 agent 工作流，让模型自己决定"用哪种语气"回复，验证是否能提升用户体验

6. Google 推出 7 个 AI 加持的暑期出行规划技巧

  • 来源: Google AI Blog
  • 分类: 产品与功能
  • 重要性: ⭐⭐ (2/5)
  • 标签: Google Search AI 出行 暑期 规划工具 消费应用

摘要: Google 介绍了 7 个借助 AI 工具规划暑期旅行的方法，覆盖行程规划、特价发现和目的地探索等场景，主要面向消费者用户。文章本质是 Google Search/Maps 等现有 AI 功能在出行垂直场景下的整合营销，技术新意有限，但反映出 Google 在持续把 LLM 能力沉淀到日常消费产品。

实践建议

  • 在 Google Search 中尝试一句话查询"为我规划一次 5 天东京家庭行程"，观察 AI Overview 给出的结构化结果
  • 对照 ChatGPT / Perplexity 等通用 AI 助手的同样请求，评估垂直集成（机票、酒店、地图）的实际价值差距
